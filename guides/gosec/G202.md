# G202

<instructions>
Detects SQL query construction using string concatenation (`+` operator). Concatenating user input into SQL queries allows SQL injection attacks where an attacker can modify query logic to read, modify, or delete data.

Use parameterized queries with placeholder values. Pass user input as query arguments, never as part of the query string.
</instructions>

<examples>
## Bad
```go
query := "SELECT * FROM products WHERE name = '" + productName + "'"
rows, _ := db.Query(query)
```

## Good
```go
rows, err := db.Query("SELECT * FROM products WHERE name = $1", productName)
if err != nil {
    return err
}
```
</examples>

<patterns>
- Use parameterized queries with placeholders instead of `+` concatenation for SQL
- Replace `strings.Join`/`strings.Builder` SQL assembly with parameterized queries
- Use parameterized WHERE clauses — for dynamic column names, use an allowlist of permitted identifiers
</patterns>

# staticcheck: S1028

<instructions>
Detects overly complex error construction patterns. This includes using `fmt.Errorf` with no format verbs (should be `errors.New`), or wrapping errors with `fmt.Errorf("%s", err)` instead of `errors.Wrap(err, "context")` from `github.com/pkg/errors`.

Use `errors.New` for static messages and `errors.Wrap` from `github.com/pkg/errors` for wrapping.
</instructions>

<examples>
## Good
```go
err := errors.New("something went wrong")
```
</examples>

<patterns>
- Replace `fmt.Errorf` without format verbs with `errors.New`
- Replace `fmt.Errorf` with `%w` wrapping to preserve the error chain
- Replace `Sprintf` + `errors.New` with direct `fmt.Errorf`
- Wrap errors with `%w` so `errors.Is`/`errors.As` work correctly
</patterns>

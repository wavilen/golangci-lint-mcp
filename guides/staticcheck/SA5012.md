# staticcheck: SA5012

<instructions>
Detects empty `if` or `else` branches, such as `if cond {} else { doSomething() }`. An empty branch with no statements is usually a sign of incomplete code or logic that should be inverted.

Invert the condition and remove the empty branch: `if !cond { doSomething() }`.
</instructions>

<examples>
## Good
```go
if found {
    process(item)
}
```
</examples>

<patterns>
- Invert the condition and remove empty `if` bodies with logic only in `else`
- Remove empty `else` bodies left after refactoring
- Implement placeholder branches or remove them
- Remove negated conditions with empty then-branches
</patterns>

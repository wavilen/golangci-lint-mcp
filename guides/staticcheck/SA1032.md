# staticcheck: SA1032

<instructions>
Detects `time.LoadLocation` called with an invalid timezone name. The timezone string must match an IANA Time Zone database entry (e.g., `"America/New_York"`, `"Europe/London"`). Common mistakes include using UTC offsets like `"UTC+5"` or abbreviations like `"PST"` which are ambiguous.
</instructions>

<examples>
## Bad
```go
loc, err := time.LoadLocation("UTC+5") // not a valid IANA timezone name
```

## Good
```go
loc, err := time.LoadLocation("Asia/Karachi")
// or for fixed offsets:
loc = time.FixedZone("UTC+5", 5*60*60)
```
</examples>

<patterns>
- Using fixed offset strings like `"UTC+5"` or `"GMT+2"` with `LoadLocation`
- Using timezone abbreviations like `"PST"`, `"EST"`, `"CET"` (ambiguous)
- Hardcoding timezone names that may not exist on minimal systems (use `time.UTC` as fallback)
</patterns>

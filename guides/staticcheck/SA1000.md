# staticcheck: SA1000

<instructions>
Detects calls to `regexp.Compile`, `regexp.MustCompile`, and related functions where the pattern string is not a valid regular expression. This causes a panic at runtime with `MustCompile` or returns an error with `Compile` that is often ignored.

Fix the regex pattern syntax. Use a raw string literal to avoid double-escaping backslashes.
</instructions>

<examples>
## Bad
```go
// Unmatched parenthesis — panics at runtime
re := regexp.MustCompile("foo(bar")
```

## Good
```go
re := regexp.MustCompile("foo\\(bar\\)")
// or use raw string to avoid double escaping
re = regexp.MustCompile(`foo\(bar\)`)
```
</examples>

<patterns>
- Unbalanced parentheses or brackets in regex patterns
- Invalid escape sequences when not using raw string literals
- Dynamically constructed regex strings with unescaped metacharacters
- Using `regexp.MustCompile` with user input that hasn't been validated
</patterns>

<related>
SA1010, gosec/G204
</related>

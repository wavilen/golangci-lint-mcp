# staticcheck: S1019

<instructions>
Detects pointless interface method declarations or expressions that have no effect. This includes using a method value without calling it, or defining interface methods that are never used or are redundant.

Either call the method, use the expression result, or remove the dead code.
</instructions>

<examples>
## Bad
```go
type Reader interface {
    Read(p []byte) (n int, err error)
    Read(p []byte) (n int, err error) // duplicate method
}
```

## Good
```go
type Reader interface {
    Read(p []byte) (n int, err error)
}
```
</examples>

<patterns>
- Call or remove method value expressions that are computed but never invoked
- Remove duplicate method declarations in interfaces
- Use or remove unused method expressions assigned to variables
- Remove interface embedding that duplicates methods
</patterns>

<related>
S1018, S1005, SA4017

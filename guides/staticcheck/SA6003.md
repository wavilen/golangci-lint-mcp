# staticcheck: SA6003

<instructions>
Detects conversions of `rune` to `string` using `string(r)` where a single rune is converted to a string. While valid Go, this allocation may be unnecessary if the rune is being compared or used in a context where the string conversion can be avoided.

If comparing single characters, compare runes directly. If converting for output, use `fmt.Fprintf` with `%c` to avoid allocation.
</instructions>

<examples>
## Good
```go
for _, r := range text {
    if r == 'A' { // rune comparison, no allocation
        count++
    }
}
```
</examples>

<patterns>
- Compare runes directly with `r == 'x'` instead of `string(r) == "x"`
- Avoid converting runes to strings for single-character comparisons
- Pass runes directly to `strings.ContainsRune` instead of string-converted runes
- Use `strings.Builder` instead of building strings from individual runes in a loop
</patterns>

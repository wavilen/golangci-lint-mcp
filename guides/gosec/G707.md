# G707

<instructions>
Detects SMTP header injection where tainted user input flows into email headers (To, From, Subject, CC). Attackers can inject additional headers via newline characters (`\r\n`), adding BCC recipients to exfiltrate data, modifying the email body, or changing the recipient list.

Validate email addresses with a strict pattern before passing them to SMTP functions. Strip or reject any `\r` or `\n` characters in header values. Use a well-tested email library that handles header construction safely.
</instructions>

<examples>
## Bad
```go
func sendMail(to, subject, body string) error {
    // Attacker input for "to": "victim@example.com\r\nBcc: attacker@evil.com"
    msg := "To: " + to + "\r\nSubject: " + subject + "\r\n\r\n" + body
    return smtp.SendMail("localhost:25", nil, "noreply@example.com", []string{to}, []byte(msg))
}
```

## Good
```go
func sendMail(to, subject, body string) error {
    // Reject any newlines in header values
    if strings.ContainsAny(to, "\r\n") || strings.ContainsAny(subject, "\r\n") {
        return fmt.Errorf("invalid characters in email header")
    }
    // Validate email format
    if !isValidEmail(to) {
        return fmt.Errorf("invalid email address")
    }
    msg := "To: " + to + "\r\nSubject: " + subject + "\r\n\r\n" + body
    return smtp.SendMail("localhost:25", nil, "noreply@example.com", []string{to}, []byte(msg))
}
```
</examples>

<patterns>
- User-supplied email addresses placed directly in SMTP message headers
- Subject lines from user input containing `\r\n` sequences
- Contact form fields (name, email, subject) used in email header construction
- Feedback or notification systems where recipient addresses come from user input
</patterns>

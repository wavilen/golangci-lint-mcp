# staticcheck: S1026

<instructions>
Detects redundant type assertions where the value is already known to be of the asserted type. This can happen after a type switch or when the assertion result is immediately used as the same type.

Remove the redundant type assertion and use the value directly.
</instructions>

<examples>
## Bad
```go
var s string = "hello"
x := s.(string) // s is already a string (not an interface)
```

## Good
```go
var s string = "hello"
x := s
```
</examples>

<patterns>
- Type asserting a concrete type to the same type
- Double type assertion after already extracting in a type switch
- Asserting x.(T) after code guarantees x is already T
- Redundant assertions on narrow interface types
</patterns>

<related>
S1016, S1036, SA4016

# staticcheck: S1001

<instructions>
Detects `for` loops that copy elements from one slice to another element-by-element. The built-in `copy()` function is more concise, clearer in intent, and often faster since it can use `memmove` internally.

Replace the loop with `copy(dst, src)`.
</instructions>

<examples>
## Bad
```go
src := []int{1, 2, 3}
dst := make([]int, len(src))
for i, v := range src {
    dst[i] = v
}
```

## Good
```go
src := []int{1, 2, 3}
dst := make([]int, len(src))
copy(dst, src)
```
</examples>

<patterns>
- Manual element-by-element slice copying with a for-range loop
- Copying a slice into a pre-allocated destination using index assignment
- Using append in a loop when copy would suffice
- Copying between slices of the same type where copy semantics apply
</patterns>

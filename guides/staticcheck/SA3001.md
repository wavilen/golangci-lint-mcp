# staticcheck: SA3001

<instructions>
Detects assignments to `benchmark.B.N` inside benchmark functions. `B.N` is set by the benchmarking framework to control the number of iterations. Assigning to it interferes with the benchmark runner and produces invalid timing results.
</instructions>

<examples>
## Bad
```go
func BenchmarkParse(b *testing.B) {
    b.N = 1000 // don't override the framework's iteration count
    for i := 0; i < b.N; i++ {
        parse(input)
    }
}
```

## Good
```go
func BenchmarkParse(b *testing.B) {
    for i := 0; i < b.N; i++ {
        parse(input)
    }
}
```
</examples>

<patterns>
- Never assign `b.N` to a fixed value — the testing framework controls iteration count
- Avoid assigning `b.N` in calculations that write back to `b.N`
- Separate benchmark iteration control from test setup logic
</patterns>

# staticcheck: SA9004

<instructions>
Detects const declarations with an unnecessary explicit type when the type can be inferred from the initializer. For example, `const x int64 = 5` is redundant because the compiler can infer the type. Removing the explicit type makes the constant more flexible — it becomes an untyped constant that can be used with any compatible type.

Remove the explicit type from the const declaration unless the type constraint is intentional.
</instructions>

<examples>
## Bad
```go
const maxSize int64 = 1024
const pi float64 = 3.14159
```

## Good
```go
const maxSize = 1024
const pi = 3.14159
```
</examples>

<patterns>
- `const x int = 0` where untyped constant suffices
- `const s string = "hello"` with no need for typed constant
- Explicitly typed const used in arithmetic with other typed values
- Package-level constants with unnecessary type annotations
</patterns>

# G504

<instructions>
Detects imports of `net/http/cgi`. The CGI protocol forks a new process for each request, making it vulnerable to resource exhaustion and injection attacks. CGI also exposes environment variables and HTTP headers in ways that are difficult to secure.

Use `net/http` directly to serve HTTP handlers in-process. Go's standard HTTP server is performant, memory-safe, and avoids the overhead and attack surface of CGI process management.
</instructions>

<examples>
## Bad
```go
import "net/http/cgi"

func main() {
    cgi.Serve(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprintf(w, "Hello from CGI")
    }))
}
```

## Good
```go
import "net/http"

func main() {
    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprintf(w, "Hello from native HTTP")
    })
    http.ListenAndServe(":8080", nil)
}
```
</examples>

<patterns>
- Remove `net/http/cgi` — use `net/http` handlers directly for Go web services
- Replace CGI deployment with native Go HTTP server — use reverse proxy instead of CGI gateway
- Replace CGI deployment with a standalone Go HTTP server behind a reverse proxy
- Avoid CGI entirely — its environment variable model leaks request metadata
</patterns>

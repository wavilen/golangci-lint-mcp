# staticcheck: SA4018

<instructions>
Detects self-assignment where a variable is assigned to itself, such as `x = x`. This is a no-op and indicates a typo, incomplete refactoring, or a merge conflict artifact.

Remove the self-assignment or fix it to assign the intended value.
</instructions>

<examples>
## Bad
```go
x := 10
x = x // no-op
```

## Good
```go
x := 10
x = 20
```
</examples>

<patterns>
- `x = x` after renaming a variable during refactoring
- Struct field assigned to itself: `s.Field = s.Field`
- Merge conflict artifacts leaving both sides identical
- Incomplete edits where the right-hand side was not updated
</patterns>

</related>

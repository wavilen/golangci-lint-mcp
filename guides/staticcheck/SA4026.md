# staticcheck: SA4026

<instructions>
Detects moderately complex dead code patterns that are not caught by simpler checks. This includes unreachable code after return/panic statements, conditions that are always true or false due to upstream logic, and branches that can never execute.

Remove the dead code or fix the control flow that makes it unreachable.
</instructions>

<examples>
## Bad
```go
func process(v int) int {
    if v > 0 {
        return v
    }
    return -v
    slog.Info("never reached") // dead code after return
}
```

## Good
```go
func process(v int) int {
    if v > 0 {
        return v
    }
    return -v
}
```
</examples>

<patterns>
- Statements after `return`, `panic`, or `continue` in the same block
- Code guarded by conditions that are always false
- Switch cases that cannot be reached due to earlier exhaustive cases
- Unreachable error handling paths
</patterns>

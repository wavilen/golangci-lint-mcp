# staticcheck: S1009

<instructions>
Detects redundant nil checks before operations that already handle nil safely. In Go, `range` over a nil slice is a no-op, `len(nil)` returns 0, and type assertions on nil interfaces panic but `x.(*T)` where x is already checked is redundant.

Remove the unnecessary nil guard.
</instructions>

<examples>
## Good
```go
for _, v := range items {
    slog.Info("value", "v", v)
}
```
</examples>

<patterns>
- Remove nil checks before `range` — nil slice range is a safe no-op
- Remove nil checks before `len()` — `len(nil)` returns 0
- Remove redundant nil checks before type assertions that are already guarded
- Eliminate double nil checks where the inner block already handles nil
</patterns>

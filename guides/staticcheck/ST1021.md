# staticcheck: ST1021

<instructions>
Detects unnecessary code patterns in variable and type declarations. This includes using `var` where `const` would be appropriate, or declaring variables that could be inlined.

Use `const` for values that never change, or inline the variable.
</instructions>

<examples>
## Good
```go
const defaultTimeout = 30 * time.Second
```
</examples>

<patterns>
- Replace `var` with `const` for values that never change
- Replace package-level `var` declarations with `const` when the value is immutable
- Use `const` for configuration values that should not be reassigned
- Convert mutable variables that are never reassigned to `const`
</patterns>

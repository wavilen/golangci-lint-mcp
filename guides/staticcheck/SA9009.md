# staticcheck: SA9009

<instructions>
Detects uses of `context.Background()` where `context.TODO()` would be more appropriate. `context.Background()` should be used in main, init, and top-level request handlers as the root context. `context.TODO()` signals that the code has not been properly wired to propagate a context yet and should be fixed later.

Use `context.TODO()` when you are unsure which context to use, and `context.Background()` only when you intentionally need a root context.
</instructions>

<examples>
## Bad
```go
func processRequest(req *Request) error {
    // should use context.TODO() or accept ctx parameter
    result, err := db.Query(context.Background(), req.Query)
    return err
}
```

## Good
```go
func processRequest(ctx context.Context, req *Request) error {
    result, err := db.Query(ctx, req.Query)
    return err
}
```
</examples>

<patterns>
- `context.Background()` in functions that should accept a `context.Context` parameter
- Using Background in middleware or handlers that already have a context available
- Tests using Background when the function under test expects a propagated context
- Library code hardcoding Background instead of accepting context from callers
</patterns>

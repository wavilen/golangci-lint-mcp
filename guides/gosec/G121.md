# G121

<instructions>
Detects unsafe Cross-Origin Resource Sharing (CORS) bypass patterns. Overly permissive CORS configurations — such as reflecting the `Origin` header or using `*` for `Access-Control-Allow-Origin` with `Access-Control-Allow-Credentials: true` — allow malicious websites to read responses from your API.

Use strict CORS policies. Only allow specific trusted origins, and avoid combining `Access-Control-Allow-Credentials: true` with wildcard origins.
</instructions>

<examples>
## Bad
```go
w.Header().Set("Access-Control-Allow-Origin", "*")
w.Header().Set("Access-Control-Allow-Credentials", "true")
```

## Good
```go
allowed := map[string]bool{"https://app.example.com": true}
origin := r.Header.Get("Origin")
if allowed[origin] {
    w.Header().Set("Access-Control-Allow-Origin", origin)
    w.Header().Set("Access-Control-Allow-Credentials", "true")
}
```
</examples>

<patterns>
- `Access-Control-Allow-Origin: *` with `Allow-Credentials: true`
- Blindly reflecting the request `Origin` header into `Allow-Origin`
- CORS middleware allowing all methods and headers without restriction
</patterns>

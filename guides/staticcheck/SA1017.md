# staticcheck: SA1017

<instructions>
Detects `time.After` used in a `select` loop. `time.After` creates a new `time.Timer` each iteration that is never stopped. If the loop runs frequently, these orphaned timers accumulate and leak memory until they fire.

Use `time.NewTimer` and `Timer.Stop` to clean up, or `time.NewTicker` for periodic checks.
</instructions>

<examples>
## Bad
```go
for {
    select {
    case <-time.After(5 * time.Second): // new timer each iteration, never stopped
        handleTimeout()
    case msg := <-ch:
        handle(msg)
    }
}
```

## Good
```go
timer := time.NewTimer(5 * time.Second)
defer timer.Stop()
for {
    select {
    case <-timer.C:
        handleTimeout()
        timer.Reset(5 * time.Second)
    case msg := <-ch:
        handle(msg)
    }
}
```
</examples>

<patterns>
- Replace `time.After` in `for`-`select` loops with `time.NewTimer` to prevent unbounded allocations
- Use `time.NewTimer` with `Reset` in long-running server loops instead of `time.After`
- Call `Timer.Stop` to clean up timers created with `time.NewTimer`
</patterns>

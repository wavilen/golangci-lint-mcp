# staticcheck: S1010

<instructions>
Detects slice expressions of the form `a[low:high]` where the high bound equals `len(a)`. Since omitting the high bound defaults to the slice length, `a[low:]` is equivalent to `a[low:len(a)]` but more concise.

Remove the redundant `len(a)` from the slice expression.
</instructions>

<examples>
## Good
```go
suffix := data[2:]
```
</examples>

<patterns>
- Replace `s[i:len(s)]` with `s[i:]` — the upper bound defaults to length
- Remove redundant three-index slicing when the cap equals length
- Omit explicit length in re-slicing when it defaults correctly
- Fix copy-pasted slice expressions that use a stale upper bound
</patterns>

# staticcheck: QF1009

<instructions>
Detects unnecessary code that can be removed without affecting behavior. This includes redundant nil checks, unnecessary assignments, or dead code that the compiler can optimize away but clutters the source.

Remove the unnecessary code entirely.
</instructions>

<examples>
## Bad
```go
if x == x {
    doSomething()
}
```

## Good
```go
doSomething()
```
</examples>

<patterns>
- Self-comparisons: x == x (always true), x != x (always false)
- Unnecessary boolean conversions in conditions
- Dead code after unconditional returns
- Redundant nil checks on guaranteed-non-nil values
</patterns>

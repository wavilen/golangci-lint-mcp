# staticcheck: SA1015

<instructions>
Detects `time.Sleep` called inside a loop. This is almost always a mistake — it blocks the goroutine unnecessarily and makes the program unresponsive. Use `time.Ticker` for repeated delays, or `time.After`/`time.NewTimer` for a single deadline.
</instructions>

<examples>
## Bad
```go
for {
    doWork()
    time.Sleep(5 * time.Second) // blocks goroutine, hard to cancel
}
```

## Good
```go
ticker := time.NewTicker(5 * time.Second)
defer ticker.Stop()
for {
    doWork()
    <-ticker.C
}
```
</examples>

<patterns>
- `time.Sleep` in `for` loops as a poor man's ticker
- Busy-wait loops with sleep instead of event-driven design
- Loop-based retry logic using `time.Sleep` instead of `time.After` with context cancellation
</patterns>

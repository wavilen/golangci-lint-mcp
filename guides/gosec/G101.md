# G101

<instructions>
Detects hardcoded credentials such as passwords, API keys, tokens, and secrets in source code. Embedding secrets in code makes them visible in version control, CI logs, and compiled binaries.

Move secrets to environment variables, secret managers, or configuration files outside the source tree. Use `os.Getenv` or a secrets management library.
</instructions>

<examples>
## Good
```go
func connect() {
    password := os.Getenv("DB_PASSWORD")
    if password == "" {
        log.Fatal("DB_PASSWORD not set")
    }
    db, _ := sql.Open("postgres", fmt.Sprintf("postgres://user:%s@db/host", password))
}
```
</examples>

<patterns>
- Avoid hardcoding credentials in variable names like `password`, `secret`, `token` — use environment variables or a secrets manager
- Remove credentials from connection strings — pass them via environment variables or runtime configuration
- Move API keys to environment variables or secret managers, never commit them to source control
- Replace real credentials in test fixtures with mock values or test-specific secrets
</patterns>

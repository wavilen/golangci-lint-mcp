# staticcheck: S1035

<instructions>
Detects `else` branches that are unnecessary because the `if` body contains a terminal statement (return, break, continue, goto, panic). When the if-block always exits, the else-block code can be de-indented to the same level.

Move the else block's code out of the else and remove the else keyword.
</instructions>

<examples>
## Bad
```go
if err != nil {
    return err
} else {
    slog.Info("success")
}
```

## Good
```go
if err != nil {
    return err
}
slog.Info("success")
```
</examples>

<patterns>
- else after return in the if body
- else after panic in the if body
- else after continue or break in the if body
- Deeper nesting caused by unnecessary else blocks
</patterns>

<related>
SA5012, S1020, S1027

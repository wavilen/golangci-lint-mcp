# G109

<instructions>
Detects when the result of `strconv.Atoi` (which returns `int`) is converted to `int32` or `int16` without checking for overflow. On 64-bit systems, `int` is 64 bits, so a large parsed value can silently overflow when narrowed.

Check the parsed value against the target type's range before converting, or use `strconv.ParseInt` with the appropriate bit size.
</instructions>

<examples>
## Bad
```go
val, _ := strconv.Atoi(input)
size := int32(val) // silent overflow if val > math.MaxInt32
```

## Good
```go
val, err := strconv.ParseInt(input, 10, 32)
if err != nil {
    return errors.Wrap(err, "invalid int32 value")
}
size := int32(val)
```
</examples>

<patterns>
- Converting `strconv.Atoi` result to `int32` or `int16` without bounds checking
- Parsing user input as int then narrowing without overflow detection
- Parsing sizes, lengths, or counts from network/external sources
</patterns>

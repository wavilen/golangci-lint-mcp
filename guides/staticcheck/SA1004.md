# staticcheck: SA1004

<instructions>
Detects calls to `time.Sleep` in test functions. Sleeping in tests makes the test suite slow and flaky — timing-dependent tests fail intermittently on slow or overloaded machines. Use channels, sync primitives, or `testify/assert.Eventually` to wait for conditions instead.
</instructions>

<examples>
## Good
```go
func TestServerReady(t *testing.T) {
    startServer()
    assert.Eventually(t, server.IsReady, 5*time.Second, 100*time.Millisecond)
}
```
</examples>

<patterns>
- Replace `time.Sleep` in tests with channel-based synchronization or `assert.Eventually`
- Replace sleep-before-assert patterns with polling for network or goroutine state
- Use condition polling instead of fixed-duration sleeps in tests
</patterns>

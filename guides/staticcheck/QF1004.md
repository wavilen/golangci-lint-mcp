# staticcheck: QF1004

<instructions>
Detects comparisons of `time.Time` values using `==` or `!=`. The `time.Time` type contains both a wall clock and a monotonic clock reading, and direct equality checks may give unexpected results. Use the `Equal` method instead, which handles monotonic clock differences correctly.

Replace `t1 == t2` with `t1.Equal(t2)`.
</instructions>

<examples>
## Bad
```go
if startTime == endTime {
    slog.Info("same time")
}
```

## Good
```go
if startTime.Equal(endTime) {
    slog.Info("same time")
}
```
</examples>

<patterns>
- Replace `==` with `Equal()` for comparing `time.Time` values
- Use `!t1.Equal(t2)` instead of `!=` for time inequality
- Avoid `==` on `time.Now()` results — use `Equal()` to handle monotonic clock
- Use `t1.Equal(t2)` instead of struct comparison on Time values
</patterns>

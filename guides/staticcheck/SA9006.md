# staticcheck: SA9006

<instructions>
Detects functions that return a nil value as a concrete type (not an interface). When a function declares a concrete return type like `*MyType` and returns `nil`, callers may expect a non-nil value. More subtly, returning `nil` for an interface type at the call site can cause surprising behavior where the interface value is non-nil (typed nil).

Return the zero value of the concrete type, or change the return type to an interface if nil is a valid result.
</instructions>

<examples>
## Bad
```go
func getWriter() *bytes.Buffer {
    if !enabled {
        return nil // caller must check for nil
    }
    return &bytes.Buffer{}
}
```

## Good
```go
func getWriter() io.Writer {
    if !enabled {
        return nil // explicit nil interface return
    }
    return &bytes.Buffer{}
}
```
</examples>

<patterns>
- Return an interface type instead of `nil` as a concrete pointer type
- Add explicit nil checks for factory functions that may not produce a value
- Ensure all code paths return a valid value when the return type is concrete
- Avoid assigning typed nil values to interface-typed variables
</patterns>

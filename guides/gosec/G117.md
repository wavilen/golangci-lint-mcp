# G117

<instructions>
Detects struct fields containing secrets (passwords, tokens, API keys) that are serialized to JSON, YAML, XML, or TOML via struct tags. Marshaling sensitive fields into output exposes them in logs, API responses, and storage.

Add `json:"-"` (or equivalent for other formats) to secret fields, or use a separate view struct for serialization that omits sensitive data.
</instructions>

<examples>
## Bad
```go
type User struct {
    Name     string `json:"name"`
    Password string `json:"password"`
    Token    string `json:"token"`
}

data, _ := json.Marshal(user)
w.Write(data) // secrets exposed in response
```

## Good
```go
type User struct {
    Name     string `json:"name"`
    Password string `json:"-"`
    Token    string `json:"-"`
}

// Or use a view struct:
type UserView struct {
    Name string `json:"name"`
}
```
</examples>

<patterns>
- Password/token fields with `json:"password"` or `json:"token"` tags
- Full models serialized to HTTP response bodies including secret fields
- Log statements marshaling entire structs that contain credentials
</patterns>

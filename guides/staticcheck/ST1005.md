# staticcheck: ST1005

<instructions>
Detects error strings that end with punctuation (periods, colons, exclamation marks, or newlines). Go convention says error strings should be standalone phrases without trailing punctuation, because they are often combined with context.

Remove trailing punctuation from error messages.
</instructions>

<examples>
## Bad
```go
return errors.New("file not found.")
```

## Good
```go
return errors.New("file not found")
```
</examples>

<patterns>
- Error messages ending with a period
- Error messages ending with a colon
- Error messages ending with an exclamation mark
- Error messages ending with punctuation that breaks when wrapped
</patterns>

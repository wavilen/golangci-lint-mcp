# G503

<instructions>
Detects imports of `crypto/rc4`. RC4 is a stream cipher with multiple known vulnerabilities, including biases in its keystream output that allow attackers to recover plaintext. It was removed from TLS in RFC 7465.

Use AES-GCM (`crypto/aes` + `crypto/cipher`) or ChaCha20-Poly1305 (`golang.org/x/crypto/chacha20poly1305`) for stream encryption. Both provide authenticated encryption.
</instructions>

<examples>
## Good
```go
import (
    "crypto/aes"
    "crypto/cipher"
    "crypto/rand"
)

func encrypt(key []byte, data []byte) ([]byte, error) {
    block, err := aes.NewCipher(key)
    if err != nil {
        return nil, fmt.Errorf("creating cipher: %w", err)
    }
    gcm, err := cipher.NewGCM(block)
    if err != nil {
        return nil, fmt.Errorf("creating GCM: %w", err)
    }
    nonce := make([]byte, gcm.NonceSize())
    _, readErr := rand.Read(nonce)
    if readErr != nil {
        return nil, fmt.Errorf("generating nonce: %w", readErr)
    }
    return gcm.Seal(nonce, nonce, data, nil), nil
}
```
</examples>

<patterns>
- Remove `crypto/rc4` imports — RC4 is broken and should never be used
- Replace RC4 in legacy protocols with AES-GCM — RC4 has known biases and vulnerabilities
- Replace custom RC4 stream ciphers with `crypto/aes` GCM or ChaCha20-Poly1305
- Eliminate RC4 entirely — key reuse in RC4 enables the Fluhrer-McGrew attack
</patterns>

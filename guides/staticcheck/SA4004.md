# staticcheck: SA4004

<instructions>
Detects loop variables captured by closures in `for` loops (pre-Go 1.22 semantics). Before Go 1.22, all iterations share a single loop variable. When a goroutine or deferred closure captures the loop variable by reference, it will see the final value from the last iteration rather than the value at the time the closure was created.

Starting with Go 1.22, loop variables are scoped per-iteration. For older code, create a local copy inside the loop body and capture that instead.
</instructions>

<examples>
## Good
```go
for _, val := range values {
    val := val // shadow with per-iteration copy
    go func() {
        process(val)
    }()
}
```
</examples>

<patterns>
- Capture loop variables by creating a local copy `val := val` before launching goroutines
- Copy iterator variables before deferring closures inside loops
- Store per-iteration copies of loop variables in closures saved during iteration
- Add `val := val` shadow for code targeting Go versions before 1.22
</patterns>

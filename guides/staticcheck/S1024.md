# staticcheck: S1024

<instructions>
Detects code that computes time until a deadline as `deadline.Sub(time.Now())` instead of using the idiomatic `time.Until(deadline)`. The `time.Until` helper is the counterpart to `time.Since` and is clearer in intent.

Replace `deadline.Sub(time.Now())` with `time.Until(deadline)`.
</instructions>

<examples>
## Bad
```go
remaining := deadline.Sub(time.Now())
```

## Good
```go
remaining := time.Until(deadline)
```
</examples>

<patterns>
- Replace `deadline.Sub(time.Now())` with `time.Until(deadline)` for time remaining
- Replace manual duration-until computation with `time.Until`
- Use `time.Until` instead of `Sub(time.Now())` as the direct equivalent
- Replace `Sub`-based timeout measurement with `time.Until`
</patterns>

# staticcheck: SA4028

<instructions>
Detects comparisons between struct values of different types using `==`. Even if two structs have the same fields, Go requires exact type matches for `==` comparison. This results in a compile error or indicates a type confusion bug.

Ensure both operands are the same type. Convert one side if the types are compatible aliases, or compare individual fields.
</instructions>

<examples>
## Bad
```go
type Point struct{ X, Y int }
type Coord struct{ X, Y int }

p := Point{1, 2}
c := Coord{1, 2}
if p == c { // type mismatch: Point != Coord
    slog.Info("equal")
}
```

## Good
```go
if Point(c) == p {
    slog.Info("equal")
}
```
</examples>

<patterns>
- Convert types before comparing structs with identical fields but different names
- Convert `type Alias int` to `int` before direct comparison
- Convert between cross-package struct types with the same layout before comparing
- Update comparisons after refactoring splits a shared type into separate types
</patterns>

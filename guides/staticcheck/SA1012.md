# staticcheck: SA1012

<instructions>
Detects `context.WithValue` calls where the key is a string. String keys are prone to collisions between packages — two packages may use the same key string and silently overwrite each other's values. Define a custom key type to avoid collisions.
</instructions>

<examples>
## Bad
```go
ctx = context.WithValue(ctx, "requestID", "abc-123")
```

## Good
```go
type contextKey string
const requestIDKey contextKey = "requestID"

ctx = context.WithValue(ctx, requestIDKey, "abc-123")
```
</examples>

<patterns>
- Using string literals as context keys across multiple packages
- Using built-in types (`int`, `string`) as context keys without wrapping in a custom type
- Defining context key constants as `string` type instead of a named type
</patterns>

<related>
SA1013, SA1029
</related>

# staticcheck: SA1032

<instructions>
Detects `time.LoadLocation` called with an invalid timezone name. The timezone string must match an IANA Time Zone database entry (e.g., `"America/New_York"`, `"Europe/London"`). Common mistakes include using UTC offsets like `"UTC+5"` or abbreviations like `"PST"` which are ambiguous.
</instructions>

<examples>
## Good
```go
loc, err := time.LoadLocation("Asia/Karachi")
// or for fixed offsets:
loc = time.FixedZone("UTC+5", 5*60*60)
```
</examples>

<patterns>
- Replace fixed offset strings with `time.FixedZone` or valid IANA names
- Replace ambiguous timezone abbreviations with IANA names
- Use `time.UTC` as a fallback for timezone names that may not exist on minimal systems
</patterns>

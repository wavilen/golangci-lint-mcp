# staticcheck: SA4023

<instructions>
Detects impossible type assertions where the asserted type can never be satisfied because it is not in the interface's type set. This is detected at compile time for interface types with a defined type set. The assertion will always panic at runtime.

Fix the type assertion to use a type that is actually in the interface's type set, or remove the dead code.
</instructions>

<examples>
## Bad
```go
var r io.Reader = strings.NewReader("hello")
_ = r.(io.Closer) // strings.Reader does not implement io.Closer
```

## Good
```go
var r io.ReadCloser = io.NopCloser(strings.NewReader("hello"))
_ = r.(io.Closer)
```
</examples>

<patterns>
- Remove type assertions for interfaces the concrete type does not satisfy
- Check type assertions against the actual method set of the value's static type
- Add missing interface satisfaction checks after refactoring
- Distinguish `.(type)` in a switch from `.(Type)` assertion — fix the correct form
</patterns>

</related>

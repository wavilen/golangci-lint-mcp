# G704

<instructions>
Detects Server-Side Request Forgery (SSRF) where tainted user input flows into HTTP request URLs. Attackers can make the server send requests to internal services, cloud metadata endpoints (e.g., `169.254.169.254`), or other hosts behind the firewall.

Validate all user-supplied URLs against an allowlist of permitted domains and schemes. Block requests to private IP ranges, localhost, and link-local addresses. Use a dedicated HTTP client with a custom DNS resolver that rejects internal addresses.
</instructions>

<examples>
## Bad
```go
func fetchURL(userURL string) ([]byte, error) {
    // Attacker supplies "http://169.254.169.254/latest/meta-data/"
    // to access cloud instance metadata
    resp, err := http.Get(userURL)
    if err != nil {
        return nil, err
    }
    defer resp.Body.Close()
    return io.ReadAll(resp.Body)
}
```

## Good
```go
func fetchURL(userURL string) ([]byte, error) {
    u, err := url.Parse(userURL)
    if err != nil {
        return nil, errors.Wrap(err, "invalid URL")
    }
    // Only allow https scheme
    if u.Scheme != "https" {
        return nil, fmt.Errorf("only https URLs are permitted")
    }
    // Validate against allowlist of permitted hosts
    if !isAllowedHost(u.Hostname()) {
        return nil, fmt.Errorf("host not allowed: %s", u.Hostname())
    }
    resp, err := http.Get(userURL)
    if err != nil {
        return nil, err
    }
    defer resp.Body.Close()
    return io.ReadAll(resp.Body)
}
```
</examples>

<patterns>
- Validate user-supplied URLs against an allowlist of permitted hosts before making HTTP requests
- Check webhook/callback URLs against an allowlist of permitted domains — reject internal/private IPs
- Restrict URL fetch features to approved domains — use an allowlist and block RFC 1918 addresses
- Limit image imports to trusted domains — validate the URL host before fetching
</patterns>

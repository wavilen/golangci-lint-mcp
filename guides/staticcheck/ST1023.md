# staticcheck: ST1023

<instructions>
Detects unnecessary code patterns in composite literals and struct initializations. This includes redundant field names, unnecessary type specifications, or patterns that could be simplified.

Remove redundant specifications and use simpler initialization.
</instructions>

<examples>
## Good
```go
_ = Config{}
```
</examples>

<patterns>
- Remove explicit zero-value initializations in struct literals
- Remove redundant composite literal nesting
- Remove unnecessary type assertions in literal values
- Remove over-specified zero-value fields in struct literals
</patterns>

staticcheck/ST1013, staticcheck/S1021, staticcheck/S1016
</related>

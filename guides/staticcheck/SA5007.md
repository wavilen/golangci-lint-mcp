# staticcheck: SA5007

<instructions>
Detects struct field tags with invalid format. Struct tags must follow the `key:"value"` convention. Common mistakes include missing quotes, missing colons, extra spaces, or malformed tag syntax that the reflect package cannot parse.

Fix the tag syntax to follow the `key:"value"` format with optional `key:"value,key2:"value2"` chaining.
</instructions>

<examples>
## Bad
```go
type Config struct {
    Name string `json:name` // missing quotes around value
}
```

## Good
```go
type Config struct {
    Name string `json:"name"`
}
```
</examples>

<patterns>
- Missing quotes in struct tag values
- Missing colon between key and value
- Extra spaces inside tag strings
- Malformed tag syntax that `reflect.StructTag.Get` cannot parse
</patterns>

# staticcheck: SA1018

<instructions>
Detects `strings.Replace` (or `bytes.Replace`) called with `n=0`, which is a no-op тАФ it replaces nothing. This is almost always a mistake where the developer intended to replace all occurrences. Pass `-1` to replace all, or `1` to replace the first occurrence.
</instructions>

<examples>
## Bad
```go
result := strings.Replace(input, "old", "new", 0) // no replacements happen
```

## Good
```go
result := strings.ReplaceAll(input, "old", "new")
// or: result := strings.Replace(input, "old", "new", -1)
```
</examples>

<patterns>
- Passing literal `0` as the count argument to `strings.Replace`
- Confusing `n=0` (no replacements) with `n=-1` (all replacements)
- Copy-paste errors where the count argument was left as zero
</patterns>

# staticcheck: SA4001

<instructions>
Detects comparisons of the form `x < 0` or `x <= 0` where `x` is an unsigned integer type. Since unsigned values are always ≥ 0, these comparisons are always false. This is usually a logic error — either the variable should be signed, or the comparison should be removed.

Either change the variable type to a signed integer, or fix the comparison to test a meaningful condition.
</instructions>

<examples>
## Good
```go
var size int32
if size < 0 {
    slog.Info("negative size")
}
```
</examples>

<patterns>
- Replace unsigned comparisons with `< 0` or `<= 0` — always false for unsigned types
- Fix loop bounds using unsigned comparison with negative constants
- Replace unsigned type with signed when negative value checks are meaningful
- Use signed integer types when negative values carry semantic meaning
</patterns>

# staticcheck: SA1020

<instructions>
Detects `exec.Cmd.StdoutPipe` (or `StderrPipe`) used where the pipe is read after `cmd.Wait` is called. `Wait` closes the pipe, so reading from it after `Wait` returns always fails. Read from the pipe before calling `Wait`, or use `cmd.Output`/`cmd.CombinedOutput` instead.
</instructions>

<examples>
## Bad
```go
cmd := exec.Command("go", "version")
pipe, _ := cmd.StdoutPipe()
cmd.Start()
cmd.Wait()
io.ReadAll(pipe) // pipe already closed by Wait
```

## Good
```go
cmd := exec.Command("go", "version")
output, _ := cmd.Output()
slog.Info("output", "value", string(output))
```
</examples>

<patterns>
- Reading from `StdoutPipe` after `cmd.Wait()` returns
- Using `StdoutPipe` when `cmd.Output()` would suffice
- Forgetting to start a goroutine to drain the pipe before calling `Wait`
</patterns>

# staticcheck: ST1001

<instructions>
Detects dot imports (`import . "pkg"`) which make the imported names appear in the current file's namespace without qualification. Dot imports hurt readability and maintainability because it's unclear where a name originates.

Use regular imports and qualify names with the package name.
</instructions>

<examples>
## Bad
```go
import . "fmt"

func main() {
    Println("hello")
}
```

## Good
```go
import "fmt"

func main() {
    fmt.Println("hello")
}
```
</examples>

<patterns>
- Using dot imports for convenience in application code
- Importing test packages with dot imports for assertion functions
- Using . "github.com/stretchr/testify/assert" in tests (acceptable but discouraged)
- Domain-specific languages embedded via dot import
</patterns>

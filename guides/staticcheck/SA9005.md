# staticcheck: SA9005

<instructions>
Detects attempts to marshal or encode unexported (lowercase) struct fields using `encoding/json`, `encoding/xml`, or similar packages. Unexported fields are invisible to the reflect-based encoders and will be silently omitted from the output.

Export the field (capitalize the first letter), or add an explicit unexported-field workaround if the field must remain private.
</instructions>

<examples>
## Bad
```go
type User struct {
    name string `json:"name"` // unexported, will not appear in JSON
    Age  int    `json:"age"`
}
```

## Good
```go
type User struct {
    Name string `json:"name"` // exported, appears in JSON
    Age  int    `json:"age"`
}
```
</examples>

<patterns>
- Struct fields with json/xml tags but lowercase first letter
- Private fields expected to appear in serialized output
- `encoding/json.Marshal` producing incomplete output due to unexported fields
- Adding tags to unexported fields thinking tags override visibility
</patterns>

# staticcheck: SA5001

<instructions>
Detects `defer file.Close()` or similar patterns where the error return from `Close` is not checked. For writable files and network connections, `Close` can fail — typically because buffered data could not be flushed. Ignoring this error can silently lose data.

Check the error from `Close` in the defer. Use a named return variable to propagate the error.
</instructions>

<examples>
## Good
```go
f, err := os.Create("output.txt")
if err != nil {
    return err
}
defer func() {
    if cerr := f.Close(); cerr != nil && err == nil {
        err = cerr
    }
}()
```
</examples>

<patterns>
- Check `Close` error in defer for writable files — use a deferred closure to capture the error
- Check response body close errors in defer when the response body error matters
- Check network connection close errors in defer
- Check `Close` errors on logging files where flush-on-close failures should be reported
</patterns>

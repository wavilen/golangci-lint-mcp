# staticcheck: SA6000

<instructions>
Detects `regexp.Compile` or `regexp.MustCompile` called inside a loop. Compiling a regex is expensive — it parses the pattern and builds a state machine. Doing this on every iteration is a significant performance waste.

Compile the regex once before the loop (at package level or function start) and reuse the compiled `*regexp.Regexp`.
</instructions>

<examples>
## Bad
```go
for _, line := range lines {
    re := regexp.MustCompile(`\d+`)
    if re.MatchString(line) {
        slog.Info("line", "content", line)
    }
}
```

## Good
```go
re := regexp.MustCompile(`\d+`)
for _, line := range lines {
    if re.MatchString(line) {
        slog.Info("line", "content", line)
    }
}
```
</examples>

<patterns>
- Move `regexp.MustCompile` out of loops — compile once at package level or before the loop
- Compile regex once and reuse it instead of calling `regexp.Compile` repeatedly
- Move regex compilation out of hot paths like HTTP handlers
- Declare regex patterns as package-level compiled variables
</patterns>

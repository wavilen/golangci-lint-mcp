# staticcheck: SA1031

<instructions>
Detects `net/http.Transport` created with `DisableKeepAlives: true` but also `MaxIdleConnsPerHost` set to a non-zero value. When keep-alives are disabled, idle connections are never reused, so setting a limit on idle connections per host has no effect. Remove the contradictory configuration.
</instructions>

<examples>
## Bad
```go
tr := &http.Transport{
    DisableKeepAlives:     true,
    MaxIdleConnsPerHost:   10, // meaningless when keep-alives disabled
}
```

## Good
```go
tr := &http.Transport{
    DisableKeepAlives: true,
}
```
</examples>

<patterns>
- Setting `MaxIdleConnsPerHost` alongside `DisableKeepAlives: true`
- Setting `MaxIdleConns` alongside `DisableKeepAlives: true`
- Copying transport configs from examples that enable keep-alives and then overriding with `DisableKeepAlives`
</patterns>

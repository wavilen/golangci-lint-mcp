# G701

<instructions>
Detects SQL injection where tainted user input flows into a SQL query string without sanitization. Attackers can manipulate the query to read, modify, or delete data, or execute administrative operations on the database.

Always use parameterized queries (prepared statements) with placeholder parameters. Never interpolate user input directly into SQL strings, even with manual escaping.
</instructions>

<examples>
## Bad
```go
func getUser(db *sql.DB, username string) (*User, error) {
    query := "SELECT * FROM users WHERE name = '" + username + "'"
    row := db.QueryRow(query)
    // Attacker input "admin'--" bypasses authentication
    // ...
}
```

## Good
```go
func getUser(db *sql.DB, username string) (*User, error) {
    query := "SELECT * FROM users WHERE name = $1"
    row := db.QueryRow(query, username)
    // ...
}
```
</examples>

<patterns>
- String concatenation with user input to build SQL queries
- `fmt.Sprintf` used to construct SQL with user-controlled values
- User input passed directly to raw SQL query builders
- Dynamic table or column names from user input (cannot be parameterized — use allowlists)
</patterns>

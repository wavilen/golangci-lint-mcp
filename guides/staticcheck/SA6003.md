# staticcheck: SA6003

<instructions>
Detects conversions of `rune` to `string` using `string(r)` where a single rune is converted to a string. While valid Go, this allocation may be unnecessary if the rune is being compared or used in a context where the string conversion can be avoided.

If comparing single characters, compare runes directly. If converting for output, use `fmt.Fprintf` with `%c` to avoid allocation.
</instructions>

<examples>
## Bad
```go
for _, r := range text {
    if string(r) == "A" { // allocates string for each rune
        count++
    }
}
```

## Good
```go
for _, r := range text {
    if r == 'A' { // rune comparison, no allocation
        count++
    }
}
```
</examples>

<patterns>
- `string(runeVar) == "x"` instead of `runeVar == 'x'`
- Converting runes to strings for single-character comparison
- Using `strings.ContainsRune` with a string-converted rune
- Building strings from individual runes in a loop instead of using `strings.Builder`
</patterns>

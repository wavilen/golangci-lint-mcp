# staticcheck: S1033

<instructions>
Detects unnecessary nil checks before `range` over slices, maps, or channels. In Go, ranging over a nil slice, map, or channel is safe — it simply performs zero iterations for slices and maps, and blocks for channels.

Remove the redundant nil guard before the range statement.
</instructions>

<examples>
## Bad
```go
if items != nil {
    for _, item := range items {
        process(item)
    }
}
```

## Good
```go
for _, item := range items {
    process(item)
}
```
</examples>

<patterns>
- Checking nil before ranging over a slice (nil slice range is a no-op)
- Checking nil before ranging over a map (nil map range is a no-op)
- Guarding range with len() > 0 check (also unnecessary)
- Defensive nil checks on slices that are guaranteed safe to range
</patterns>

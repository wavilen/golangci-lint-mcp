# G203

<instructions>
Detects use of unescaped or unsanitized data in HTML templates. When `html/template` receives values that aren't properly typed (e.g., passing `template.HTML(string)` when the string contains user input), the auto-escaping is bypassed, enabling Cross-Site Scripting (XSS).

Let `html/template` auto-escape values by passing plain strings. Avoid wrapping user-controlled strings in `template.HTML`, `template.JS`, or `template.URL` types.
</instructions>

<examples>
## Bad
```go
t := template.Must(template.New("page").Parse(`<div>{{.Content}}</div>`))
t.Execute(w, map[string]any{
    "Content": template.HTML(userInput), // bypasses auto-escaping
})
```

## Good
```go
t := template.Must(template.New("page").Parse(`<div>{{.Content}}</div>`))
t.Execute(w, map[string]any{
    "Content": userInput, // auto-escaped by html/template
})
```
</examples>

<patterns>
- Avoid `template.HTML()` with user input — use `html/template` auto-escaping instead
- Remove `template.JS` and `template.URL` wrappers for user content — let the template engine escape automatically
- Switch from `text/template` to `html/template` for any HTML output to enable automatic escaping
</patterns>

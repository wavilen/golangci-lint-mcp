# G204

<instructions>
Detects use of `os/exec.Command`, `os/exec.CommandContext`, `exec.Cmd` methods, or `syscall.Exec` with user-supplied arguments. This can lead to command injection where an attacker executes arbitrary system commands.

Avoid shell commands when possible. If required, never pass user input through a shell (`/bin/sh -c`). Use `exec.Command` with separate arguments instead of concatenated strings. Validate and sanitize all inputs against an allowlist.
</instructions>

<examples>
## Bad
```go
out, _ := exec.Command("sh", "-c", "ls "+userDir).Output()
```

## Good
```go
out, err := exec.Command("ls", validatedDir).Output()
if err != nil {
    return err
}
```
</examples>

<patterns>
- Avoid shell execution with user input — pass arguments directly to `exec.Command` without `sh -c`
- Validate user-supplied command arguments against an allowlist before passing to `exec.Command`
- Replace `syscall.Exec` with `exec.Command` — validate all dynamic paths against an allowlist
</patterns>

# G107

<instructions>
Detects when a URL is constructed from user-supplied input (taint source) and passed to `http.Get`, `http.Post`, or `http.NewRequest`. This can enable Server-Side Request Forgery (SSRF) attacks where an attacker makes the server issue requests to internal services.

Validate and sanitize URLs against an allowlist of permitted domains/schemes before making HTTP requests. Use allowlists over denylists.
</instructions>

<examples>
## Bad
```go
func handler(w http.ResponseWriter, r *http.Request) {
    url := r.URL.Query().Get("url")
    resp, _ := http.Get(url)
    defer resp.Body.Close()
}
```

## Good
```go
var allowedHosts = map[string]bool{"api.example.com": true, "cdn.example.com": true}

func handler(w http.ResponseWriter, r *http.Request) {
    urlStr := r.URL.Query().Get("url")
    u, err := url.Parse(urlStr)
    if err != nil || u.Scheme != "https" || !allowedHosts[u.Hostname()] {
        http.Error(w, "invalid URL", http.StatusBadRequest)
        return
    }
    resp, _ := http.Get(u.String())
    defer resp.Body.Close()
}
```
</examples>

<patterns>
- User-controlled query parameters used directly in `http.Get` URLs
- Request body fields used to construct outbound HTTP request URLs
- URL path parameters forwarded to internal service calls without validation
</patterns>

# staticcheck: S1002

<instructions>
Detects code that uses `for i := 0; i < len(s); i++` when iterating over a slice or string. This pattern is error-prone and less readable than using `for range`.

Replace with a `for range` loop, or use the appropriate index-based access pattern.
</instructions>

<examples>
## Bad
```go
items := []string{"a", "b", "c"}
for i := 0; i < len(items); i++ {
    slog.Info("item", "value", items[i])
}
```

## Good
```go
items := []string{"a", "b", "c"}
for i, v := range items {
    slog.Info("item", "index", i, "value", v)
}
```
</examples>

<patterns>
- C-style for loops over slice indices when range is clearer
- Manual index management where range handles it automatically
- Accessing string bytes with manual index instead of range
- Omitting the value in range when only the index is needed
</patterns>

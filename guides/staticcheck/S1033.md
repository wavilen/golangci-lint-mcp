# staticcheck: S1033

<instructions>
Detects unnecessary nil checks before `range` over slices, maps, or channels. In Go, ranging over a nil slice, map, or channel is safe — it simply performs zero iterations for slices and maps, and blocks for channels.

Remove the redundant nil guard before the range statement.
</instructions>

<examples>
## Good
```go
for _, item := range items {
    process(item)
}
```
</examples>

<patterns>
- Remove nil checks before `range` over slices — nil slice range is a safe no-op
- Remove nil checks before `range` over maps — nil map range is a safe no-op
- Remove `len() > 0` guards before `range` — also unnecessary
- Remove defensive nil checks on slices that are guaranteed safe to range
</patterns>

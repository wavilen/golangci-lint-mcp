# staticcheck: S1032

<instructions>
Detects uses of `sort.Slice` with a comparison function where a typed sort function would be simpler and more efficient. Go's `sort` package provides `sort.Ints`, `sort.Strings`, `sort.Float64s`, and their `IsSorted` variants for common types.

Use the typed sort function instead of `sort.Slice` with a manual less function.
</instructions>

<examples>
## Bad
```go
sort.Slice(nums, func(i, j int) bool {
    return nums[i] < nums[j]
})
```

## Good
```go
sort.Ints(nums)
```
</examples>

<patterns>
- Using sort.Slice with x[i] < x[j] for int slices — use sort.Ints
- Using sort.Slice for string slices — use sort.Strings
- Using sort.Slice for float64 slices — use sort.Float64s
- Implementing custom Less when standard sort functions suffice
</patterns>

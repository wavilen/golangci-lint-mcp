# G110

<instructions>
Detects `io.Copy` used when decompressing data from a potentially malicious source. An attacker can craft compressed data that decompresses to a much larger size (zip bomb), causing memory exhaustion or disk fill.

Use `io.CopyN` with a reasonable size limit, or use `io.LimitedReader` to cap the decompressed output size.
</instructions>

<examples>
## Bad
```go
gz, _ := gzip.NewReader(resp.Body)
io.Copy(dst, gz) // unbounded decompression
```

## Good
```go
gz, _ := gzip.NewReader(resp.Body)
limited := io.LimitReader(gz, 10*1024*1024) // cap at 10MB
io.Copy(dst, limited)
```
</examples>

<patterns>
- `io.Copy` with gzip/zlib/bzip2 decompression without size limits
- Decompressing uploaded archive files without output size caps
- HTTP response bodies decompressed without limiting total bytes
</patterns>

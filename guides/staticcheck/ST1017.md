# staticcheck: ST1017

<instructions>
Detects unnecessary explicit type conversions where the compiler can infer the type. This includes converting between numeric types in const declarations or in contexts where the target type is known.

Remove the explicit type conversion and let the compiler handle it.
</instructions>

<examples>
## Bad
```go
const x = int(42)
```

## Good
```go
const x = 42
```
</examples>

<patterns>
- Explicit type conversion in const declarations
- Unnecessary int(), string(), or float64() conversions
- Type conversion in assignment to a typed variable
- Redundant conversions in function call arguments
</patterns>

<related>
S1016, ST1013, S1036

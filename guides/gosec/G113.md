# G113

<instructions>
Detects HTTP request smuggling vulnerabilities caused by conflicting `Content-Length` and `Transfer-Encoding` headers, or bare LF line endings in HTTP body parsing. These ambiguities can cause frontend and backend servers to interpret request boundaries differently, enabling cache poisoning or authentication bypass.

Avoid manually constructing HTTP headers that include both `Content-Length` and `Transfer-Encoding`. Let Go's `net/http` package manage headers automatically.
</instructions>

<examples>
## Good
```go
req, _ := http.NewRequest("POST", url, body)
// Let net/http manage Content-Length/Transfer-Encoding automatically
req.Header.Set("Content-Type", "application/json")
```
</examples>

<patterns>
- Avoid setting both `Content-Length` and `Transfer-Encoding` — let the Go HTTP library manage headers
- Use `http.NewRequest` instead of raw string concatenation for HTTP request construction
- Sanitize and normalize headers in custom proxy implementations — strip ambiguous headers before forwarding
</patterns>

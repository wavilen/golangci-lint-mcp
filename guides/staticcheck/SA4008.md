# staticcheck: SA4008

<instructions>
Detects a loop variable that is shadowed by a variable declared inside the loop body with the same name. The inner variable hides the outer loop variable, making the loop variable's value inaccessible within that scope. This is almost always unintentional.

Rename the inner variable to avoid shadowing, or use the loop variable directly without redeclaring it.
</instructions>

<examples>
## Bad
```go
for i := 0; i < n; i++ {
    i := i + 1 // shadows loop variable i
    slog.Info("index", "i", i) // prints modified value, loop counter unaffected
}
```

## Good
```go
for i := 0; i < n; i++ {
    next := i + 1
    slog.Info("next value", "value", next)
}
```
</examples>

<patterns>
- `x := x` pattern inside loops shadowing the iterator
- Variable declared with same name as loop counter inside loop body
- Accidental `:=` instead of `=` for inner scope variables
- Closure captures that unintentionally shadow loop variables
</patterns>

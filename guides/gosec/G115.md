# G115

<instructions>
Detects integer overflow risks when converting between numeric types of different sizes. Converting a larger type (e.g., `int64`) to a smaller type (e.g., `int32`, `int16`) without bounds checking silently truncates the value, producing incorrect results.

Use `strconv.ParseInt` with the correct bit size for parsing, or check bounds before narrowing conversions. For unsigned-to-signed conversions, verify the value fits the target range.
</instructions>

<examples>
## Good
```go
var bigVal int64 = 1 << 40
if bigVal > math.MaxInt32 || bigVal < math.MinInt32 {
    return fmt.Errorf("value %d overflows int32", bigVal)
}
smallVal := int32(bigVal)
```
</examples>

<patterns>
- Check values against target type bounds (`math.MaxInt32`, etc.) before narrowing integer conversions
- Validate unsigned-to-signed conversions — ensure the value fits in the positive range of the target type
- Guard `len()` and `cap()` casts to narrower types — add bounds checks when values exceed target range
- Validate externally sourced integer values against target type limits before conversion
</patterns>

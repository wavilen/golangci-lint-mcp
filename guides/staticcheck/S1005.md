# staticcheck: S1005

<instructions>
Detects unnecessary blank identifier assignments. This includes receiving from a closed channel into a blank identifier, or assigning to `_` when the value is never used. These patterns add noise without benefit.

Remove the unnecessary blank assignment or restructure the code.
</instructions>

<examples>
## Good
```go
<-ch      // receive and discard directly
compute() // just call the function
```
</examples>

<patterns>
- Remove unnecessary `_ =` assignments when the side effect is the only purpose
- Replace `<-ch` into blank identifier with a direct receive `<-ch`
- Use the two-value map lookup `v, ok := m[k]` instead of discarding the boolean
- Remove `_ =` before function calls where the side effect is all that matters
</patterns>

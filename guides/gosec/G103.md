# G103

<instructions>
Audits the use of `unsafe` package operations (`unsafe.Pointer`, `unsafe.Slice`, `unsafe.String`, `unsafe.Sizeof` in pointer arithmetic). The `unsafe` package bypasses Go's type safety and memory guarantees, which can lead to memory corruption, data races, and undefined behavior.

Avoid `unsafe` operations unless absolutely necessary for performance-critical code or Cgo interop. When unavoidable, document the safety invariants clearly.
</instructions>

<examples>
## Bad
```go
import "unsafe"

func bytesToString(b []byte) string {
    return *(*string)(unsafe.Pointer(&b))
}
```

## Good
```go
func bytesToString(b []byte) string {
    return string(b)
}
```
</examples>

<patterns>
- Avoid `unsafe.Pointer` for type conversions — use typed casts or `encoding/json` for safe serialization
- Replace pointer arithmetic with slice indexing — use `reflect.SliceHeader` only when absolutely necessary
- Guard `unsafe.Slice` calls with explicit length validation against the backing array size
- Remove `unsafe` byte/string conversions — Go compiler optimizes these automatically since 1.20
</patterns>

# staticcheck: SA4019

<instructions>
Detects duplicate keys in map literals. When a map literal contains the same key multiple times, only the last value for that key is retained. This is almost always a mistake caused by copy-paste or incomplete refactoring.

Remove the duplicate key or correct the key to the intended unique value.
</instructions>

<examples>
## Bad
```go
m := map[string]int{
    "a": 1,
    "b": 2,
    "a": 3, // duplicate key, overwrites "a": 1
}
```

## Good
```go
m := map[string]int{
    "a": 1,
    "b": 2,
    "c": 3,
}
```
</examples>

<patterns>
- Copy-paste errors where key was not updated
- Case-sensitive keys where different casing was intended
- Large map literals where duplicates are hard to spot visually
- Constants used as keys that resolve to the same value
</patterns>

<related>
SA4000, SA4018
</related>

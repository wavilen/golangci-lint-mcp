# staticcheck: SA5005

<instructions>
Detects `len(string)` used as a check for string emptiness or character count when the string might contain non-ASCII characters. `len(string)` returns the byte count, not the character (rune) count. For strings containing CJK characters or emojis, byte count and rune count differ significantly.

If checking for emptiness, compare with `""`. If counting characters, use `utf8.RuneCountInString(s)` or `len([]rune(s))`.
</instructions>

<examples>
## Bad
```go
if len(name) > 10 {
    slog.Info("name too long")
}
// "日本語日本語" has 6 runes but 18 bytes
```

## Good
```go
if utf8.RuneCountInString(name) > 10 {
    slog.Info("name too long")
}
```
</examples>

<patterns>
- Using `len(s)` to validate string length for user input
- Comparing `len(s)` against a character limit for internationalized text
- Truncating strings based on byte count instead of rune count
- Form validation with byte-based length checks
</patterns>

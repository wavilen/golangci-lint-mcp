# staticcheck: ST1020

<instructions>
Detects unnecessary patterns in function and method declarations. This includes declaring methods with pointer receivers when a value receiver would suffice, or patterns that add complexity without benefit.

Use the most appropriate receiver type for the method.
</instructions>

<examples>
## Bad
```go
func (s *Server) Name() string {
    return s.name // doesn't modify s, value receiver OK
}
```

## Good
```go
func (s Server) Name() string {
    return s.name
}
```
</examples>

<patterns>
- Use value receivers on methods that do not modify the receiver
- Use pointer receivers on methods that need to modify state
- Keep receiver types consistent across methods on the same type
- Remove unnecessary pointer dereferences in method bodies
</patterns>

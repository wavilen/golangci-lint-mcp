# staticcheck: S1018

<instructions>
Detects self-assignment where a variable is assigned to itself, such as `x = x`. This is a no-op and likely indicates a typo or incomplete refactoring.

Remove the self-assignment or fix it to assign the intended value.
</instructions>

<examples>
## Bad
```go
name := "Alice"
name = name // no-op
```

## Good
```go
name := "Alice"
name = "Bob"
```
</examples>

<patterns>
- Assigning a variable to itself: x = x
- Self-assignment in struct methods: s.field = s.field
- Copy-paste errors where the right-hand side was meant to be different
- Leftover debugging or partial refactoring
</patterns>

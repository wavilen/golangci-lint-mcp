# staticcheck: S1002

<instructions>
Detects code that uses `for i := 0; i < len(s); i++` when iterating over a slice or string. This pattern is error-prone and less readable than using `for range`.

Replace with a `for range` loop, or use the appropriate index-based access pattern.
</instructions>

<examples>
## Good
```go
items := []string{"a", "b", "c"}
for i, v := range items {
    slog.Info("item", "index", i, "value", v)
}
```
</examples>

<patterns>
- Replace C-style `for i := 0; i < len(s); i++` with `for range`
- Use `range` instead of manual index management over slices
- Replace manual index access on strings with `for range`
- Use `for i := range s` when only the index is needed
</patterns>

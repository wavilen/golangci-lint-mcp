# G602

<instructions>
Detects slice access patterns where the bounds may be incorrectly calculated, leading to out-of-range panics at runtime. This commonly occurs when slice bounds are derived from arithmetic on `len()` or when a slice is resliced after truncation without updating the bound variables.

Always validate slice bounds before access, prefer reslicing with `[:n]` syntax where `n` is directly checked against `len()`, and avoid complex arithmetic on slice indices.
</instructions>

<examples>
## Bad
```go
func getChunk(data []byte, offset, size int) []byte {
    end := offset + size
    // No bounds check: offset or end may exceed len(data)
    return data[offset:end]
}
```

## Good
```go
func getChunk(data []byte, offset, size int) ([]byte, error) {
    end := offset + size
    if offset < 0 || end > len(data) || offset > end {
        return nil, fmt.Errorf("slice bounds [%d:%d] out of range [0:%d]", offset, end, len(data))
    }
    return data[offset:end], nil
}
```
</examples>

<patterns>
- Validate computed slice indices against `len()` before reslicing
- Check for integer overflow in offset + size calculations before using as slice bounds
- Ensure `n` in `[:n]` does not exceed the slice's length before truncating
- Validate user-controlled slice bounds against the actual slice length before slicing
</patterns>

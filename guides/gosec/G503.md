# G503

<instructions>
Detects imports of `crypto/rc4`. RC4 is a stream cipher with multiple known vulnerabilities, including biases in its keystream output that allow attackers to recover plaintext. It was removed from TLS in RFC 7465.

Use AES-GCM (`crypto/aes` + `crypto/cipher`) or ChaCha20-Poly1305 (`golang.org/x/crypto/chacha20poly1305`) for stream encryption. Both provide authenticated encryption.
</instructions>

<examples>
## Bad
```go
import "crypto/rc4"

func encrypt(key, data []byte) ([]byte, error) {
    cipher, err := rc4.NewCipher(key)
    if err != nil {
        return nil, err
    }
    dst := make([]byte, len(data))
    cipher.XORKeyStream(dst, data)
    return dst, nil
}
```

## Good
```go
import (
    "crypto/aes"
    "crypto/cipher"
    "crypto/rand"
)

func encrypt(key []byte, data []byte) ([]byte, error) {
    block, err := aes.NewCipher(key)
    if err != nil {
        return nil, err
    }
    gcm, err := cipher.NewGCM(block)
    if err != nil {
        return nil, err
    }
    nonce := make([]byte, gcm.NonceSize())
    if _, err := rand.Read(nonce); err != nil {
        return nil, err
    }
    return gcm.Seal(nonce, nonce, data, nil), nil
}
```
</examples>

<patterns>
- `crypto/rc4` imported for any encryption
- RC4 used in legacy protocols (WEP, early TLS)
- Custom stream cipher implementations based on RC4
- RC4 key reuse across multiple messages
</patterns>

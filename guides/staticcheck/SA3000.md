# staticcheck: SA3000

<instructions>
Detects a `TestMain` function that does not call `os.Exit`. When you define `TestMain`, Go no longer automatically exits the process — you must call `os.Exit` with the return code from `m.Run()`. Without it, the test binary returns exit code 0 even if tests fail.
</instructions>

<examples>
## Bad
```go
func TestMain(m *testing.M) {
    setup()
    m.Run() // exit code ignored, process returns 0 always
}
```

## Good
```go
func TestMain(m *testing.M) {
    setup()
    code := m.Run()
    teardown()
    os.Exit(code)
}
```
</examples>

<patterns>
- `TestMain` that calls `m.Run()` without capturing or passing the exit code
- `TestMain` with setup/teardown that returns normally instead of calling `os.Exit`
- Missing `os.Exit` when `TestMain` was added to set up global test state
</patterns>

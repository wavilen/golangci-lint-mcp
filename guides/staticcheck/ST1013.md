# staticcheck: ST1013

<instructions>
Detects explicit type specifications in composite literals where the type can be inferred from the context. For example, specifying the type in a map literal key when the variable's type already constrains it.

Remove the explicit type specification and let the compiler infer it.
</instructions>

<examples>
## Bad
```go
m := map[string]int{
    "a": int(1),
}
```

## Good
```go
m := map[string]int{
    "a": 1,
}
```
</examples>

<patterns>
- Explicit type in map literal values when type is known
- Explicit type conversion in slice literals with known element type
- Redundant type specification in struct field assignments
- Unnecessary type annotations in return statements
</patterns>

<related>
S1016, S1036, S1021

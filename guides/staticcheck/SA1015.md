# staticcheck: SA1015

<instructions>
Detects `time.Sleep` called inside a loop. This is almost always a mistake — it blocks the goroutine unnecessarily and makes the program unresponsive. Use `time.Ticker` for repeated delays, or `time.After`/`time.NewTimer` for a single deadline.
</instructions>

<examples>
## Bad
```go
for {
    doWork()
    time.Sleep(5 * time.Second) // blocks goroutine, hard to cancel
}
```

## Good
```go
ticker := time.NewTicker(5 * time.Second)
defer ticker.Stop()
for {
    doWork()
    <-ticker.C
}
```
</examples>

<patterns>
- Replace `time.Sleep` in loops with `time.NewTicker` for periodic execution
- Replace busy-wait loops with sleep with event-driven design using channels
- Replace loop-based retry with `time.After` and context cancellation
</patterns>

# staticcheck: SA5002

<instructions>
Detects infinite `for` loops with no mechanism to exit ‚Äî no `break`, `return`, or `panic` inside the loop body, and the condition is always `true`. While intentional infinite loops exist (e.g., server main loops), most cases are bugs where an exit condition was forgotten.

Add an exit condition, a `break`, or a `return` statement. If the loop is intentionally infinite, consider using a channel or context for graceful shutdown.
</instructions>

<examples>
## Bad
```go
for {
    process()
    // no break, return, or exit condition
}
```

## Good
```go
for {
    select {
    case <-ctx.Done():
        return
    case msg := <-ch:
        process(msg)
    }
}
```
</examples>

<patterns>
- Add an exit condition to `for {}` loops ‚Äî use `break`, `return`, or a condition
- Check a done channel in queue-processing loops for shutdown
- Add a shutdown mechanism to server loops using context or done channel
- Fix accidental infinite loops from missing or removed exit conditions
</patterns>

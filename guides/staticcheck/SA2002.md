# staticcheck: SA2002

<instructions>
Detects `testing.T.FailNow` (or `Fatal`, `Fatalf`, `Skip`, `Skipf`) called from a goroutine other than the test goroutine. These functions call `runtime.Goexit`, which only stops the current goroutine — not the test. The test continues running, and the failure may be missed or cause panics.

Use `t.Error`/`t.Errorf` to record failures from other goroutines and communicate back to the test goroutine via channels or sync primitives.
</instructions>

<examples>
## Good
```go
func TestParallel(t *testing.T) {
    valueCh := make(chan int)
    errCh := make(chan string, 1)
    go func() {
        val := <-valueCh
        if val != 42 {
            errCh <- fmt.Sprintf("expected 42, got %d", val)
            return
        }
        errCh <- ""
    }()
    valueCh <- 42
    if msg := <-errCh; msg != "" {
        t.Error(msg)
    }
}
```
</examples>

<patterns>
- Replace `t.Fatal` or `t.Fatalf` inside a goroutine with channel-based error reporting
- Avoid `t.FailNow` in goroutines processing test subtasks — use `t.Error` instead
- Call `t.Skip` from the main test goroutine, never from spawned goroutines
</patterns>

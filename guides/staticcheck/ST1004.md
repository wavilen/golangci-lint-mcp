# staticcheck: ST1004

<instructions>
Detects uses of `fmt.Errorf` or `errors.New` where the error message starts with a capital letter. By Go convention, error messages should not start with a capital letter because they may appear mid-sentence when wrapped.

Lowercase the first letter of the error message string.
</instructions>

<examples>
## Bad
```go
return fmt.Errorf("Invalid input: %d", n)
```

## Good
```go
return fmt.Errorf("invalid input: %d", n)
```
</examples>

<patterns>
- Lowercase the first letter of error messages in `fmt.Errorf` and `errors.New`
- Lowercase error strings in `errors.New` calls
- Lowercase the first letter of format strings in `fmt.Errorf`
- Write error messages as lowercase phrases not standalone capitalized sentences
</patterns>

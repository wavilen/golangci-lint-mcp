# staticcheck: SA4019

<instructions>
Detects duplicate keys in map literals. When a map literal contains the same key multiple times, only the last value for that key is retained. This is almost always a mistake caused by copy-paste or incomplete refactoring.

Remove the duplicate key or correct the key to the intended unique value.
</instructions>

<examples>
## Bad
```go
m := map[string]int{
    "a": 1,
    "b": 2,
    "a": 3, // duplicate key, overwrites "a": 1
}
```

## Good
```go
m := map[string]int{
    "a": 1,
    "b": 2,
    "c": 3,
}
```
</examples>

<patterns>
- Fix copy-paste errors in map literals by updating duplicated keys
- Ensure case-sensitive map keys use the intended casing
- Review large map literals for duplicate keys that are hard to spot visually
- Check that constants used as keys do not resolve to the same value
</patterns>

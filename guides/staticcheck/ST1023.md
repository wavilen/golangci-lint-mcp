# staticcheck: ST1023

<instructions>
Detects unnecessary code patterns in composite literals and struct initializations. This includes redundant field names, unnecessary type specifications, or patterns that could be simplified.

Remove redundant specifications and use simpler initialization.
</instructions>

<examples>
## Bad
```go
_ = Config{Field1: Config{}.Field1}
```

## Good
```go
_ = Config{}
```
</examples>

<patterns>
- Initializing struct fields to their zero values explicitly
- Redundant composite literal nesting
- Unnecessary type assertions in literal values
- Over-specifying zero-value fields in struct literals
</patterns>

<related>
ST1013, S1021, S1016

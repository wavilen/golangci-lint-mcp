# staticcheck: SA4010

<instructions>
Detects `append` calls whose result is not assigned back to a variable. Since `append` may allocate a new backing array, discarding the result means the original slice is unchanged and the appended elements are lost. This is always a bug.

Assign the result of `append` back to the same variable: `s = append(s, x)`.
</instructions>

<examples>
## Bad
```go
items := []string{"a", "b"}
append(items, "c") // result discarded, items unchanged
```

## Good
```go
items := []string{"a", "b"}
items = append(items, "c")
```
</examples>

<patterns>
- `append(slice, x)` without assigning result
- Appending to a slice stored in a map: `append(m[key], x)` without reassigning `m[key]`
- Chained operations where `append` return is lost
- Confusing `append` with methods that mutate in-place
</patterns>

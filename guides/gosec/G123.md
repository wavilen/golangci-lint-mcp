# G123

<instructions>
Detects TLS session resumption configurations that may bypass `VerifyPeerCertificate` when `VerifyConnection` is not also set. When a TLS session is resumed, Go skips the `VerifyPeerCertificate` callback, potentially accepting a previously-rejected certificate.

When using `VerifyPeerCertificate`, also set `VerifyConnection` to ensure custom verification runs on resumed sessions. Alternatively, disable session resumption with `tls.Config.SessionTicketsDisabled = true`.
</instructions>

<examples>
## Good
```go
tlsConfig := &tls.Config{
    InsecureSkipVerify: true,
    VerifyPeerCertificate: func(rawCerts [][]byte, chains [][]*x509.Certificate) error {
        return verifyCert(chains)
    },
    VerifyConnection: func(cs tls.ConnectionState) error {
        return verifyCert(cs.VerifiedChains)
    },
}
```
</examples>

<patterns>
- Add `VerifyConnection` alongside `VerifyPeerCertificate` — ensure verification covers session resumption
- Set both `VerifyPeerCertificate` and `VerifyConnection` to prevent bypass during TLS session resumption
- Remove `InsecureSkipVerify` and implement proper certificate verification with both peer and connection checks
</patterns>

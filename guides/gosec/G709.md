# G709

<instructions>
Detects unsafe deserialization where tainted user input flows into `encoding/gob`, `encoding/json` into `any`, or similar deserialization without type constraints. Attackers can craft payloads that instantiate unexpected types, trigger method side effects, or cause denial of service through deeply nested structures.

Always deserialize into concrete, well-defined struct types. Avoid `json.Unmarshal` into `any` or `interface{}` when the source is untrusted. Set size limits on input and use `json.Decoder` with `DisallowUnknownFields()` for stricter parsing.
</instructions>

<examples>
## Bad
```go
func handlePayload(data []byte) error {
    var payload any
    // Untrusted data deserialized into unconstrained interface{}
    // Attacker can send arbitrary JSON structures
    if err := json.Unmarshal(data, &payload); err != nil {
        return err
    }
    // Using payload without type assertion is dangerous
    return nil
}
```

## Good
```go
func handlePayload(data []byte) error {
    var payload struct {
        Action  string `json:"action"`
        Target  string `json:"target"`
        Payload string `json:"payload"`
    }
    dec := json.NewDecoder(bytes.NewReader(data))
    dec.DisallowUnknownFields()
    // Limit input size
    if len(data) > maxPayloadSize {
        return fmt.Errorf("payload too large")
    }
    if err := dec.Decode(&payload); err != nil {
        return err
    }
    // Validate fields against allowlist
    if !isValidAction(payload.Action) {
        return fmt.Errorf("invalid action: %s", payload.Action)
    }
    return nil
}
```
</examples>

<patterns>
- `json.Unmarshal` into `any` or `interface{}` with untrusted input
- `gob.Decode` with attacker-controlled data and registered types
- Deeply nested JSON causing stack overflow during parsing
- Type confusion attacks where deserialized type differs from expected type
</patterns>

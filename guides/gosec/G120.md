# G120

<instructions>
Detects unbounded `ParseMultipartForm` calls in HTTP handlers without a `MaxMultipartMemory` limit. An attacker can upload an arbitrarily large multipart form, exhausting server memory.

Set `http.Server.MaxMultipartMemory` or call `r.ParseMultipartForm(maxBytes)` with a reasonable limit before processing uploads.
</instructions>

<examples>
## Good
```go
func uploadHandler(w http.ResponseWriter, r *http.Request) {
    if err := r.ParseMultipartForm(32 << 20); err != nil { // 32MB limit
        http.Error(w, "invalid upload", http.StatusBadRequest)
        return
    }
    file, _, err := r.FormFile("upload")
    if err != nil {
        http.Error(w, "upload failed", http.StatusBadRequest)
        return
    }
    defer file.Close()
}
```
</examples>

<patterns>
- Set a reasonable max memory limit in `ParseMultipartForm` — avoid passing `0` (unlimited)
- Add `http.MaxBytesReader` before reading request bodies to enforce upload size limits
- Wrap request body with `http.MaxBytesReader` to cap the total bytes read from the client
</patterns>

# G304

<instructions>
Detects file paths constructed from user-supplied input (taint source). This enables path traversal attacks where an attacker uses `../` sequences to read or write files outside the intended directory.

Sanitize and validate file paths. Use `filepath.Clean` and verify the resolved path is within the allowed base directory. Never trust user input for file paths directly.
</instructions>

<examples>
## Bad
```go
filename := r.URL.Query().Get("file")
data, _ := os.ReadFile("/var/data/" + filename) // path traversal via ../
```

## Good
```go
basePath := "/var/data/"
filename := r.URL.Query().Get("file")
fullPath := filepath.Clean(filepath.Join(basePath, filename))
if !strings.HasPrefix(fullPath, basePath) {
    http.Error(w, "invalid path", http.StatusBadRequest)
    return
}
data, err := os.ReadFile(fullPath)
```
</examples>

<patterns>
- Sanitize user input before joining with file paths — use `filepath.Join` and validate the result stays within the allowed root
- Validate `r.FormValue` against an allowlist or use `filepath.SafeJoin` before file system operations
- Check file paths from HTTP headers against a base directory — reject paths containing `..` or absolute paths
</patterns>

# staticcheck: ST1009

<instructions>
Detects unnecessary error variables that could be inlined. This occurs when an error is assigned to a variable but only used once, or when the error could be returned directly without the intermediate variable.

Inline the error creation or return it directly.
</instructions>

<examples>
## Good
```go
if err := validate(input); err != nil {
    return err
}
```
</examples>

<patterns>
- Inline error variables used only in immediate if checks
- Inline multi-line error creation where a single-line form suffices
- Combine error declaration and check into a single if statement
- Remove redundant error intermediate variables
</patterns>

# staticcheck: ST1020

<instructions>
Detects unnecessary patterns in function and method declarations. This includes declaring methods with pointer receivers when a value receiver would suffice, or patterns that add complexity without benefit.

Use the most appropriate receiver type for the method.
</instructions>

<examples>
## Bad
```go
func (s *Server) Name() string {
    return s.name // doesn't modify s, value receiver OK
}
```

## Good
```go
func (s Server) Name() string {
    return s.name
}
```
</examples>

<patterns>
- Pointer receiver on methods that don't modify the receiver
- Value receiver on methods that should modify state
- Inconsistent receiver types across methods on the same type
- Unnecessary pointer dereference in method bodies
</patterns>

<related>
ST1006, ST1007, gocritic/typeDefFirst

# G601

<instructions>
Detects taking the address of a range loop variable, which captures a reference to a single variable that is reused across iterations. By the time the reference is used, the variable may hold the value from the last iteration. In Go versions before 1.22, this is a common source of subtle bugs.

In Go 1.22+, range loop variables are scoped per-iteration by default, eliminating this issue. For older Go versions, create a local copy of the loop variable inside the loop body before taking its address.
</instructions>

<examples>
## Bad
```go
// Go < 1.22: all pointers point to the last element
var pointers []*int
for _, v := range values {
    pointers = append(pointers, &v) // address of loop variable
}
```

## Good
```go
// Option 1: local copy (works in all Go versions)
var pointers []*int
for _, v := range values {
    v := v // shadow with new variable per iteration
    pointers = append(pointers, &v)
}

// Option 2: use index directly
for i := range values {
    pointers = append(pointers, &values[i])
}
```
</examples>

<patterns>
- Use a local copy of the range variable before taking its address: `v := v; slice = append(slice, &v)`
- Pass the loop variable as a parameter to the goroutine: `go func(val T) { ... }(v)`
- Add a local copy before appending the address: `v := v; slice = append(slice, &v)`
- Bind the range variable to a local copy inside the closure: `v := v; go func() { use(v) }()`
</patterns>

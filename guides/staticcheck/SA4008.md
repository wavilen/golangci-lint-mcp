# staticcheck: SA4008

<instructions>
Detects a loop variable that is shadowed by a variable declared inside the loop body with the same name. The inner variable hides the outer loop variable, making the loop variable's value inaccessible within that scope. This is almost always unintentional.

Rename the inner variable to avoid shadowing, or use the loop variable directly without redeclaring it.
</instructions>

<examples>
## Good
```go
for i := 0; i < n; i++ {
    next := i + 1
    slog.Info("next value", "value", next)
}
```
</examples>

<patterns>
- Rename inner variables that shadow the loop iterator to avoid `x := x` patterns
- Avoid redeclaring variables with the same name as the loop counter inside the body
- Fix accidental `:=` instead of `=` for inner scope variable assignments
- Rename closure variables that unintentionally shadow loop variables
</patterns>

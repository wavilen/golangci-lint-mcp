# G104

<instructions>
Detects unchecked error return values. When a function returns an error and the caller discards it with `_`, failures go silently unnoticed, potentially causing data corruption, crashes, or security vulnerabilities downstream.

Always check error return values. If intentionally ignoring, assign to `_` with an accompanying comment explaining why.
</instructions>

<examples>
## Bad
```go
os.MkdirAll("/data/dir", 0755)
data, _ := os.ReadFile("config.json")
```

## Good
```go
if err := os.MkdirAll("/data/dir", 0755); err != nil {
    return errors.Wrap(err, "creating data directory")
}
data, err := os.ReadFile("config.json")
if err != nil {
    return errors.Wrap(err, "reading config")
}
```
</examples>

<patterns>
- Check all error returns from file I/O operations — handle or propagate errors from `os.Open`, `os.Create`
- Handle errors from `fmt.Fprintf` on `http.ResponseWriter` — a write error means the client disconnected
- Validate error returns from `json.Marshal` and `json.Unmarshal` before using the result
- Check `io.Copy` error return — partial writes and client disconnections need handling
</patterns>

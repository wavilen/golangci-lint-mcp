# staticcheck: SA2000

<instructions>
Detects `sync.WaitGroup.Add` called inside the goroutine being waited on instead of before launching it. Calling `Add` inside the goroutine creates a race: the `WaitGroup` counter may still be zero when `Wait` is called, causing `Wait` to return prematurely before the goroutine starts work.

Call `wg.Add(1)` before the `go` statement, not inside the goroutine function.
</instructions>

<examples>
## Good
```go
var wg sync.WaitGroup
for _, item := range items {
    wg.Add(1) // before launching goroutine
    go func(it string) {
        defer wg.Done()
        process(it)
    }(item)
}
wg.Wait()
```
</examples>

<patterns>
- Call `wg.Add(1)` before the `go` statement — not inside the goroutine function
- Place `wg.Add` and `go func` in the correct order — they are not interchangeable
- Replace loop-body patterns like `go func() { wg.Add(1); ... }()` with `wg.Add(1)` before `go func()`
</patterns>

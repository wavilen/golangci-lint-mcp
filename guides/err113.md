# err113

<instructions>
Err113 (errcheck-style rule from go-err113) detects runtime errors created with `errors.New` or `fmt.Errorf` at package or function level, and checks that exported errors follow Go conventions. It also flags direct comparisons of errors with `==` instead of using `errors.Is`.

Define sentinel errors as package-level `var` values and use `errors.Is` for comparison.
</instructions>

<examples>
## Bad
```go
if err == io.EOF {
    return ""
}
```

## Good
```go
if errors.Is(err, io.EOF) {
    return ""
}
```
</examples>

<patterns>
- Use `errors.Is(err, target)` instead of direct `err ==` comparisons to support wrapped errors
- Declare sentinel errors as package-level `var` values, not inside function bodies
- Ensure sentinel errors used across packages are exported (capitalize the `var` name)
- Use `%w` verb in `fmt.Errorf` when wrapping errors to enable `errors.Is`/`errors.As` matching
</patterns>

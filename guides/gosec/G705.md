# G705

<instructions>
Detects Cross-Site Scripting (XSS) where tainted user input flows into HTML output without proper escaping. Attackers can inject `<script>` tags or event handlers that execute JavaScript in other users' browsers, enabling session theft, data exfiltration, or phishing.

Always HTML-escape user input before rendering it in HTML templates. Use Go's `html/template` package which auto-escapes by default. Never use `text/template` for HTML output.
</instructions>

<examples>
## Bad
```go
import "text/template"

func renderProfile(w http.ResponseWriter, username string) {
    // Attacker input "<script>alert(document.cookie)</script>"
    // executes in the victim's browser
    tmpl := template.Must(template.New("profile").Parse(
        "<h1>Welcome, " + username + "</h1>"))
    tmpl.Execute(w, nil)
}
```

## Good
```go
import "html/template"

func renderProfile(w http.ResponseWriter, username string) {
    // html/template auto-escapes HTML entities
    tmpl := template.Must(template.New("profile").Parse(
        "<h1>Welcome, {{.}}</h1>"))
    tmpl.Execute(w, username)
}
```
</examples>

<patterns>
- User input rendered in HTML via `text/template` instead of `html/template`
- String concatenation of user input into HTML responses
- User input placed in JavaScript contexts without JSON encoding
- URL parameters reflected directly in HTML pages
</patterns>

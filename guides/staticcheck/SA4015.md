# staticcheck: SA4015

<instructions>
Detects calls to `math.Pow` with an integer exponent, such as `math.Pow(x, 2)` or `math.Pow(x, 3)`. Using `math.Pow` for small integer powers is slower and less precise than simple multiplication. It also forces conversion to/from `float64`, which can lose precision for large integers.

Replace with direct multiplication: `x * x` for squares, `x * x * x` for cubes.
</instructions>

<examples>
## Good
```go
result := x * x
```
</examples>

<patterns>
- Replace `math.Pow(x, 2)` with `x * x` for squaring
- Replace `math.Pow(x, 3)` with `x * x * x` for cubing
- Avoid converting integers to float64 just to use `math.Pow`
- Use direct multiplication in performance-sensitive code instead of `math.Pow`
</patterns>

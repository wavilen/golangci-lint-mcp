# staticcheck: SA9001

<instructions>
Detects `defer` statements inside `for range` loops. Each iteration adds a new deferred function, but none execute until the enclosing function returns. For large collections, this accumulates many deferred calls, consuming memory and delaying resource cleanup.

Process each item in a helper function so defer runs per iteration, or use explicit cleanup instead of defer.
</instructions>

<examples>
## Bad
```go
func processAll(files []string) error {
    for _, f := range files {
        file, err := os.Open(f)
        if err != nil {
            return err
        }
        defer file.Close() // all files stay open until processAll returns
        process(file)
    }
    return nil
}
```

## Good
```go
func processAll(files []string) error {
    for _, f := range files {
        if err := processOne(f); err != nil {
            return err
        }
    }
    return nil
}

func processOne(name string) error {
    file, err := os.Open(name)
    if err != nil {
        return err
    }
    defer file.Close() // closed after each file
    return process(file)
}
```
</examples>

<patterns>
- Extract loop body into a helper function so `defer` runs per iteration instead of accumulating
- Avoid deferring file closes inside loops — use a helper function for each file
- Avoid deferring mutex unlocks inside loops — restructure with helper functions
- Avoid deferring response body closes in URL iteration — process each URL in a helper
</patterns>

# staticcheck: SA1029

<instructions>
Detects `context.WithValue` called with a key whose type doesn't match the expected type for retrieval via `context.Value`. If you store with a key of type `myKey` but retrieve with a key of type `string`, the lookup returns `nil`. The key type must be identical — not just the key value.
</instructions>

<examples>
## Bad
```go
ctx := context.WithValue(context.Background(), "user", "alice")
// Somewhere else, using a different key type:
type keyT string
val := ctx.Value(keyT("user")) // nil — keyT != string
```

## Good
```go
type keyT string
const userKey keyT = "user"
ctx := context.WithValue(context.Background(), userKey, "alice")
val := ctx.Value(userKey) // "alice"
```
</examples>

<patterns>
- Ensure context key type matches between `WithValue` and `Value` calls
- Use named constants consistently
- Update all `Value` call sites when refactoring context key types
</patterns>

<related>
SA1012, SA1013
</related>

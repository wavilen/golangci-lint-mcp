# staticcheck: ST1009

<instructions>
Detects unnecessary error variables that could be inlined. This occurs when an error is assigned to a variable but only used once, or when the error could be returned directly without the intermediate variable.

Inline the error creation or return it directly.
</instructions>

<examples>
## Bad
```go
err := validate(input)
if err != nil {
    return err
}
```

## Good
```go
if err := validate(input); err != nil {
    return err
}
```
</examples>

<patterns>
- Error assigned to variable but only used in immediate if check
- Multi-line error creation where inline would suffice
- Error variable declared before a single check
- Redundant error intermediate variable
</patterns>

<related>
S1005, S1035, SA1006

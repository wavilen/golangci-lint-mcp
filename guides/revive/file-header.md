# revive: file-header

<instructions>
Enforces that every source file contains a required header comment (e.g., copyright notice, license, or author information). Teams use this to ensure legal and attribution requirements are met across the codebase.

Add the required header comment block at the top of the file, before the package declaration. Configure the expected header format in your revive config.
</instructions>

<examples>
## Bad
```go
package handler

import "net/http"
```

## Good
```go
// Copyright 2025 Acme Corp. All rights reserved.
// Use of this source code is governed by the MIT license.

package handler

import "net/http"
```
</examples>

<patterns>
- New files created without the standard copyright header
- Files generated by tools or scaffolding that omit the header
- Files where the header was accidentally removed during editing
- Inconsistent header formats across the codebase
- Missing license headers in open-source projects
</patterns>

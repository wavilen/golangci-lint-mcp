# staticcheck: SA4033

<instructions>
Detects `net.Conn.Close` called in a `defer` without checking its return value. While closing a connection after use is correct, the `Close` method can return an error indicating data was not fully flushed or the connection was already broken.

Check the error from `Close`, especially for connections being written to where data loss is possible.
</instructions>

<examples>
## Good
```go
conn, err := net.Dial("tcp", "example.com:80")
if err != nil {
    return err
}
defer func() {
    if err := conn.Close(); err != nil {
        log.Printf("close error: %v", err)
    }
}()
```
</examples>

<patterns>
- Check the error return from `defer conn.Close()` in a deferred function
- Check error returns from HTTP response body closes in defer
- Check error returns from file closes in defer statements
- Check close errors during network connection error handling paths
</patterns>

# G119

<instructions>
Detects unsafe HTTP redirect policies that may propagate sensitive headers (Authorization, Cookie) across host boundaries. When `CheckRedirect` is customized or disabled, credentials intended for one host can leak to a different host during redirects.

Do not disable redirect checking. When customizing redirect policy, strip sensitive headers before following cross-host redirects.
</instructions>

<examples>
## Bad
```go
client := &http.Client{
    CheckRedirect: func(req *http.Request, via []*http.Request) error {
        return http.ErrUseLastResponse // skips redirect validation
    },
}
```

## Good
```go
client := &http.Client{
    CheckRedirect: func(req *http.Request, via []*http.Request) error {
        if len(via) >= 10 {
            return fmt.Errorf("too many redirects")
        }
        if req.URL.Host != via[0].URL.Host {
            req.Header.Del("Authorization")
            req.Header.Del("Cookie")
        }
        return nil
    },
}
```
</examples>

<patterns>
- Setting `CheckRedirect` to `nil` or returning `http.ErrUseLastResponse` indiscriminately
- Custom redirect handlers that forward all headers without host validation
- `http.Transport` with disabled redirect following combined with manual redirect loops
</patterns>

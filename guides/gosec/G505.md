# G505

<instructions>
Detects imports of `crypto/sha1`. SHA-1 is broken for collision resistance — practical collision attacks have been demonstrated (SHAttered, 2017). It is unsuitable for digital signatures, certificates, or any context where collision resistance is required.

Use `crypto/sha256` or `crypto/sha512` for hashing. If you need SHA-1 only for non-security purposes (e.g., Git object hashes, legacy compatibility), document that the usage is intentional and non-cryptographic.
</instructions>

<examples>
## Bad
```go
import "crypto/sha1"

func fingerprint(data []byte) []byte {
    h := sha1.Sum(data)
    return h[:]
}
```

## Good
```go
import "crypto/sha256"

func fingerprint(data []byte) []byte {
    h := sha256.Sum256(data)
    return h[:]
}
```
</examples>

<patterns>
- `crypto/sha1` imported for hashing in security contexts
- SHA-1 used in certificate generation or verification
- SHA-1 used for data integrity checks where tampering is a concern
- Git-style content addressing (intentional non-cryptographic use)
</patterns>

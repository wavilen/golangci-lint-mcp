# staticcheck: SA1028

<instructions>
Detects `net.LookupHost` used to resolve a hostname. `LookupHost` returns a slice of IP address strings (not hostnames), which can be confusing. If you need `net.IP` values, use `net.LookupIP` directly. If you need both hostnames and IPs, use `net.LookupAddr` for reverse lookups.
</instructions>

<examples>
## Bad
```go
addrs, _ := net.LookupHost("example.com")
for _, a := range addrs {
    ip := net.ParseIP(a) // manual parsing of string to IP
    slog.Info("ip address", "value", ip)
}
```

## Good
```go
ips, _ := net.LookupIP("example.com")
for _, ip := range ips {
    slog.Info("ip address", "value", ip)
}
```
</examples>

<patterns>
- Using `LookupHost` then manually parsing strings to `net.IP`
- Confusing the string output of `LookupHost` with hostname records
- Using `LookupHost` when `LookupIP` provides the desired type directly
</patterns>

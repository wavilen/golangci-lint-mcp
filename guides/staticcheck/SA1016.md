# staticcheck: SA1016

<instructions>
Detects misuse of `time.Timer.Reset` where the timer channel is not drained before calling `Reset`. If the timer already fired and the channel contains a value, `Reset` will not clear it, causing the next receive to read a stale value from before the reset.

Drain the channel with a non-blocking read before calling `Reset`: `if !t.Stop() { <-t.C }`.
</instructions>

<examples>
## Good
```go
t := time.NewTimer(time.Second)
<-t.C
if !t.Stop() {
    <-t.C // drain stale value
}
t.Reset(2 * time.Second)
```
</examples>

<patterns>
- Call `Stop` and drain the channel before calling `Reset` on a `time.Timer`
- Drain the channel before `Reset`
- Follow the drain protocol when reusing timers in loops
</patterns>

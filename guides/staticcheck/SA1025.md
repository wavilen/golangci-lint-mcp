# staticcheck: SA1025

<instructions>
Detects misuse of `time.Timer.Stop`. A common mistake is assuming `Stop` prevents the timer from firing if it has already sent on its channel. `Stop` returns `false` when the timer already fired, meaning the channel has a value that must be drained. Ignoring this return value causes the next `Reset` or channel read to behave unexpectedly.

Check the return value of `Stop` and drain the channel if it returns `false`.
</instructions>

<examples>
## Bad
```go
t := time.NewTimer(time.Second)
t.Stop()
// If timer already fired, t.C has an unread value
t.Reset(2 * time.Second)
```

## Good
```go
t := time.NewTimer(time.Second)
if !t.Stop() {
    <-t.C // drain the fired timer
}
t.Reset(2 * time.Second)
```
</examples>

<patterns>
- Check the return value of `Timer.Stop()` and drain `t.C` if it returns `false`
- Drain `t.C` after `Stop` returns `false` to clear stale timer values
- Guard `Reset` with the drain pattern to drain before resetting
</patterns>

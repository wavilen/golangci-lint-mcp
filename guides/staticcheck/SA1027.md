# staticcheck: SA1027

<instructions>
Detects `atomic.Value.Store` calls where the concrete type differs from the type of the first value stored. `atomic.Value` requires all stored values to have the same concrete type — the first `Store` or `Load` sets the type permanently. Mixing types causes a panic at runtime.
</instructions>

<examples>
## Bad
```go
var v atomic.Value
v.Store("hello")
v.Store(42) // panic: Store of inconsistent type
```

## Good
```go
var v atomic.Value
v.Store("hello")
v.Store("world") // same concrete type: string
```
</examples>

<patterns>
- Storing values of different types in the same `atomic.Value` across code paths
- First store being implicit (via `Load` returning nil) and subsequent stores using different types
- Using `atomic.Value` for heterogeneous data instead of type-safe alternatives
</patterns>

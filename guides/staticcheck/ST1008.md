# staticcheck: ST1008

<instructions>
Detects functions that return values in a non-standard position or where the error return is not the last return value. By Go convention, error should always be the last return value, and boolean success values should come before errors.

Reorder return values so error is last.
</instructions>

<examples>
## Bad
```go
func Lookup(key string) (error, string) {
    // error should be last return value
}
```

## Good
```go
func Lookup(key string) (string, error) {
    // error as last return is standard Go convention
}
```
</examples>

<patterns>
- Move the error return value to the last position in function signatures
- Place `context.Context` as the first function parameter
- Place boolean success values before error returns
- Follow standard Go parameter ordering conventions
</patterns>

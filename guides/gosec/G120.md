# G120

<instructions>
Detects unbounded `ParseMultipartForm` calls in HTTP handlers without a `MaxMultipartMemory` limit. An attacker can upload an arbitrarily large multipart form, exhausting server memory.

Set `http.Server.MaxMultipartMemory` or call `r.ParseMultipartForm(maxBytes)` with a reasonable limit before processing uploads.
</instructions>

<examples>
## Bad
```go
func uploadHandler(w http.ResponseWriter, r *http.Request) {
    r.ParseMultipartForm(0) // unlimited memory usage
    file, _, _ := r.FormFile("upload")
    defer file.Close()
}
```

## Good
```go
func uploadHandler(w http.ResponseWriter, r *http.Request) {
    r.ParseMultipartForm(32 << 20) // 32MB limit
    file, _, err := r.FormFile("upload")
    if err != nil {
        http.Error(w, "upload failed", http.StatusBadRequest)
        return
    }
    defer file.Close()
}
```
</examples>

<patterns>
- `ParseMultipartForm(0)` or `ParseMultipartForm` with no memory limit
- File upload handlers without size restrictions
- Missing `http.MaxBytesReader` wrapper before reading request body
</patterns>

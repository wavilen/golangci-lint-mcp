# staticcheck: SA5003

<instructions>
Detects `defer` statements inside `if` bodies. While `defer` is legal inside any block, a `defer` in an `if` still defers until the *function* returns, not until the `if` block exits. This is often confusing and rarely the intended behavior.

Move the `defer` to the outer function scope, or use a separate function to limit the defer scope.
</instructions>

<examples>
## Good
```go
func process(path string) error {
    if cond {
        if err := processFile(path); err != nil {
            return err
        }
    }
    return nil
}

func processFile(path string) error {
    f, err := os.Open(path)
    if err != nil {
        return err
    }
    defer f.Close() // deferred until processFile() returns
    return nil
}
```
</examples>

<patterns>
- Move `defer` out of `if` blocks — defers run when the function returns, not when the block exits
- Extract conditional resource cleanup into a separate function for proper defer scoping
- Move `defer mu.Unlock()` to the outer scope — not inside an `if` block
- Move deferred logging or tracing out of conditionals unless the conditional function scope is intentional
</patterns>

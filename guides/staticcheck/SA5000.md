# staticcheck: SA5000

<instructions>
Detects assignments to entries in a nil map. Writing to a nil map causes a runtime panic in Go. This happens when a map variable is declared but never initialized with `make` or a map literal.

Initialize the map before writing to it using `make(map[K]V)` or a map literal `map[K]V{}`.
</instructions>

<examples>
## Bad
```go
var m map[string]int
m["key"] = 42 // panic: assignment to entry in nil map
```

## Good
```go
m := make(map[string]int)
m["key"] = 42
```
</examples>

<patterns>
- Declaring a map with `var m map[K]V` without initializing it
- Maps declared at package level without `make`
- Pointer to map where the map is nil when dereferenced
- Struct fields of map type that are never initialized
</patterns>

# staticcheck: SA5004

<instructions>
Detects `for { select {} }` patterns where the select has no exit condition. The select block loops forever without a way to break out of the outer for loop. This is a common pattern for consuming channels but the lack of a done channel or context cancellation check means the goroutine can never be stopped.

Add a `case <-ctx.Done()` or `case <-done:` branch to the select to allow graceful shutdown.
</instructions>

<examples>
## Bad
```go
for {
    select {
    case msg := <-ch:
        process(msg)
    // no exit condition
    }
}
```

## Good
```go
for {
    select {
    case <-ctx.Done():
        return ctx.Err()
    case msg := <-ch:
        process(msg)
    }
}
```
</examples>

<patterns>
- Add a `case <-ctx.Done()` branch to `for { select {} }` loops for graceful shutdown
- Add a shutdown signal channel to event loops
- Make worker goroutines cancellable with context or done channel
- Add `ctx.Done()` case to long-running select loops
</patterns>

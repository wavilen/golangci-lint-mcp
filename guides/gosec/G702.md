# G702

<instructions>
Detects command injection where tainted user input flows into arguments of `os/exec` commands. Attackers can inject shell metacharacters or additional commands to execute arbitrary programs on the host.

Avoid passing user input to shell commands. If unavoidable, validate input against a strict allowlist of permitted characters or values. Never use `sh -c` with user input. Pass arguments individually to `exec.Command` rather than concatenating into a single string.
</instructions>

<examples>
## Good
```go
func ping(host string) ([]byte, error) {
    // Validate input against allowed pattern
    if !isValidHost(host) {
        return nil, fmt.Errorf("invalid host: %s", host)
    }
    // Arguments passed individually — no shell interpretation
    return exec.Command("ping", "-c", "1", host).Output()
}

func isValidHost(h string) bool {
    // Strict allowlist: IP address or DNS name
    matched, _ := regexp.MatchString(`^[a-zA-Z0-9.\-]+$`, h)
    return matched
}
```
</examples>

<patterns>
- Avoid `sh -c` with user input — pass arguments directly to `exec.Command` as separate parameters
- Remove shell invocation from `exec.Command` — pass arguments as separate strings without shell interpretation
- Validate file paths from user input against an allowlist before passing to `exec.Command`
- Sanitize the environment passed to `exec.Command` — remove user-controlled `PATH` or `LD_PRELOAD`
</patterns>

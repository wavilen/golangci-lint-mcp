# G112

<instructions>
Detects when `http.Server` is created without setting `ReadHeaderTimeout`. Missing this timeout allows clients to hold connections open indefinitely by sending headers slowly (Slowloris attack), exhausting server resources.

Always set `ReadHeaderTimeout` on `http.Server`. A reasonable default is 5-10 seconds.
</instructions>

<examples>
## Bad
```go
srv := &http.Server{Addr: ":8080"}
srv.ListenAndServe()
```

## Good
```go
srv := &http.Server{
    Addr:              ":8080",
    ReadHeaderTimeout: 5 * time.Second,
    ReadTimeout:       10 * time.Second,
    IdleTimeout:       120 * time.Second,
}
srv.ListenAndServe()
```
</examples>

<patterns>
- Set `ReadHeaderTimeout` on every `http.Server` instance to prevent slowloris attacks
- Configure `ReadTimeout`, `WriteTimeout`, and `IdleTimeout` on custom `http.Server` instances
- Replace default server configurations with hardened timeouts for production deployments
</patterns>

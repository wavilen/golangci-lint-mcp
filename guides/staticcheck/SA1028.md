# staticcheck: SA1028

<instructions>
Detects `net.LookupHost` used to resolve a hostname. `LookupHost` returns a slice of IP address strings (not hostnames), which can be confusing. If you need `net.IP` values, use `net.LookupIP` directly. If you need both hostnames and IPs, use `net.LookupAddr` for reverse lookups.
</instructions>

<examples>
## Good
```go
ips, _ := net.LookupIP("example.com")
for _, ip := range ips {
    slog.Info("ip address", "value", ip)
}
```
</examples>

<patterns>
- Use `net.LookupIP` instead of `LookupHost` + manual string-to-IP parsing
- Distinguish `LookupHost` (returns IP address strings) from hostname records
- Replace `LookupHost` with `LookupIP` when `net.IP` values are needed directly
</patterns>

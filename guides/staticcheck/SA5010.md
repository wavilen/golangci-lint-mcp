# staticcheck: SA5010

<instructions>
Detects impossible type assertions where the type being asserted cannot possibly match because the static type of the expression is already a concrete type that does not satisfy the asserted interface. The assertion will always fail at runtime.

Use a variable with the correct interface type, or remove the impossible assertion.
</instructions>

<examples>
## Bad
```go
var r *os.File
_ = r.(io.Closer) // *os.File is already concrete, assertion trivial or wrong
```

## Good
```go
var r io.Reader = os.Stdin
if c, ok := r.(io.Closer); ok {
    c.Close()
}
```
</examples>

<patterns>
- Asserting an interface on a variable with a concrete static type
- Type assertions that always succeed or always fail
- Misunderstanding when type assertions are useful
- Switching on concrete types instead of interface types
</patterns>

<related>
SA4020, SA4023
</related>

# staticcheck: SA2001

<instructions>
Detects an empty critical section — a mutex `Lock` immediately followed by `Unlock` with no code in between. An empty critical section is dead code that adds unnecessary contention and suggests a logic error where shared data access was meant to be protected but was accidentally omitted.
</instructions>

<examples>
## Bad
```go
mu.Lock()
mu.Unlock() // no shared data access between Lock and Unlock
```

## Good
```go
mu.Lock()
sharedMap[key] = value
mu.Unlock()
```
</examples>

<patterns>
- Remove `Lock`/`Unlock` pairs with no statements between them — add the shared data access or remove the mutex calls
- Guard against refactored code where the protected operation was moved out but mutex calls remain
- Check conditional locking where the critical section was accidentally removed
</patterns>

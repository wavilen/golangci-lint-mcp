# staticcheck: SA4012

<instructions>
Detects the `!` (not) operator applied to a channel value. Channels are never boolean — applying `!` to a channel is a type error in most cases or indicates confusion with `select`. This code will not compile or does not do what the author intended.

Use a `select` statement to check channel readiness, or a boolean variable instead of a channel for the condition.
</instructions>

<examples>
## Bad
```go
ch := make(chan int)
if !ch { // channels are not boolean
    slog.Info("unreachable")
}
```

## Good
```go
ch := make(chan int)
select {
case <-ch:
    slog.Info("received")
default:
    slog.Info("no data")
}
```
</examples>

<patterns>
- `!ch` where `ch` is a channel variable
- Confusing a channel with a done/cancel boolean flag
- Attempting to check if a channel is open using boolean negation
- Mixing up `context.Done()` channel with a boolean cancel flag
</patterns>

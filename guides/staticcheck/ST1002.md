# staticcheck: ST1002

<instructions>
Detects inconsistent or poor whitespace usage in type declarations. This includes extra spaces in struct definitions, inconsistent spacing around field types, and other style issues in type declarations.

Follow standard Go formatting conventions for type declarations.
</instructions>

<examples>
## Bad
```go
type Point struct{X , Y int}
```

## Good
```go
type Point struct {
    X, Y int
}
```
</examples>

<patterns>
- Missing spaces after commas in field lists
- Inconsistent spacing in struct literal keys
- Multi-line struct definitions on a single line
- Extra whitespace around type names in declarations
</patterns>

<related>
ST1001, ST1003, gofmt

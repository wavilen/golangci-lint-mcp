# G402

<instructions>
Detects insecure TLS configuration such as `InsecureSkipVerify: true`, use of deprecated TLS versions (TLS 1.0, 1.1), or missing `MinVersion` settings. These weaken or disable certificate verification, enabling man-in-the-middle attacks.

Set `MinVersion: tls.VersionTLS12` (or `tls.VersionTLS13`) and never use `InsecureSkipVerify: true` in production. Always verify server certificates.
</instructions>

<examples>
## Bad
```go
tlsConfig := &tls.Config{
    InsecureSkipVerify: true,
}
```

## Good
```go
tlsConfig := &tls.Config{
    MinVersion: tls.VersionTLS12,
    // InsecureSkipVerify defaults to false — certificates are verified
}
```
</examples>

<patterns>
- `InsecureSkipVerify: true` disabling certificate chain verification
- Missing `MinVersion` allowing downgrade to TLS 1.0/1.1
- Custom `tls.Config` without explicit cipher suite restrictions
</patterns>

# staticcheck: SA4031

<instructions>
Detects impossible interface intersections in type assertions or interface compositions. When two interfaces have mutually exclusive method sets, no type can satisfy both simultaneously. The intersection is empty, making the assertion or composition meaningless.

Fix the interface composition or type assertion to use compatible interfaces.
</instructions>

<examples>
## Bad
```go
// No type can implement both io.Reader and io.Writer
// if the composition demands contradictory behavior
var v interface {
    MethodA()
    MethodB()
}
_ = v.(interface{ MethodC() }) // MethodC not in type set
```

## Good
```go
var v interface {
    MethodA()
    MethodB()
}
if w, ok := v.(interface{ MethodA() }); ok {
    w.MethodA()
}
```
</examples>

<patterns>
- Interface compositions with conflicting method signatures
- Type assertions to interfaces not in the value's type set
- Embedding interfaces with overlapping but incompatible methods
- Generics constraints that create empty type sets
</patterns>

<related>
SA4020, SA4023
</related>

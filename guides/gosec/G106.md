# G106

<instructions>
Detects use of `ssh.InsecureIgnoreHostKey()` which disables SSH host key verification. This makes the connection vulnerable to man-in-the-middle attacks because the client cannot verify the server's identity.

Use a known hosts file with `knownhosts.New()` or implement a custom `HostKeyCallback` that verifies the server's public key against a trusted set.
</instructions>

<examples>
## Bad
```go
config := &ssh.ClientConfig{
    User:            "deploy",
    HostKeyCallback: ssh.InsecureIgnoreHostKey(),
}
client, _ := ssh.Dial("tcp", "server:22", config)
```

## Good
```go
hostKeyCallback, _ := knownhosts.New("/home/user/.ssh/known_hosts")
config := &ssh.ClientConfig{
    User:            "deploy",
    HostKeyCallback: hostKeyCallback,
}
client, _ := ssh.Dial("tcp", "server:22", config)
```
</examples>

<patterns>
- Using `ssh.InsecureIgnoreHostKey()` in SSH client configuration
- Development SSH connections that skip host verification and get promoted to production
- Automated deployment scripts that disable host key checking for convenience
</patterns>

# staticcheck: SA4006

<instructions>
Detects situations where a value of array type (not slice) is used in a context that requires it to be addressable, such as taking its address with `&` or calling a method with a pointer receiver. Go arrays are values — copying large arrays is expensive and taking the address of a temporary is not allowed.

Use a pointer to the array, or convert to a slice. Alternatively, assign the array to a variable first to make it addressable.
</instructions>

<examples>
## Bad
```go
type Matrix [1000][1000]float64

func (m *Matrix) Transpose() {}
// cannot take address of composite literal directly in some contexts
Matrix{{1, 2}, {3, 4}}.Transpose()
```

## Good
```go
m := Matrix{{1, 2}, {3, 4}}
m.Transpose()
```
</examples>

<patterns>
- Calling pointer-receiver methods on array composite literals
- Taking the address of an array literal `&[3]int{1, 2, 3}` in older Go versions
- Passing large arrays by value instead of by pointer
- Range over large arrays causing implicit copies
</patterns>

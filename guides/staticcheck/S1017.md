# staticcheck: S1017

<instructions>
Detects calls to `len()` on an unbuffered channel. For unbuffered channels, `cap()` always returns 0 and `len()` always returns 0. Using `len()` on a channel is misleading — it suggests the value might be non-zero.

Remove the `len()` call or reconsider the channel design.
</instructions>

<examples>
## Bad
```go
ch := make(chan int)
if len(ch) > 0 { // always false for unbuffered channel
    <-ch
}
```

## Good
```go
ch := make(chan int, 10) // buffered if you need to check len
if len(ch) > 0 {
    <-ch
}
```
</examples>

<patterns>
- Remove `len()` calls on unbuffered channels — always returns 0
- Remove `cap()` calls on unbuffered channels — always returns 0
- Replace channel-length guards with buffered channels or direct receive
- Remove `len`-based guards on unbuffered channel operations that never trigger
</patterns>

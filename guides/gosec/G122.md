# G122

<instructions>
Detects Time-of-Check-to-Time-of-Use (TOCTOU) race conditions in `filepath.Walk` and `filepath.WalkDir` callbacks. If the callback checks a file's properties (existence, permissions, size) and then operates on it, an attacker can modify the filesystem between the check and the operation.

Use atomic operations that combine check and use. Open files directly with `os.Open` and handle errors rather than checking existence first with `os.Stat`. Use `O_EXCL` for creating files exclusively.
</instructions>

<examples>
## Good
```go
filepath.Walk(dir, func(path string, info os.FileInfo, err error) error {
    f, err := os.Open(path)
    if err != nil {
        return err // handle atomically: open is the check
    }
    defer f.Close()
    // process file handle, not the path
    return nil
})
```
</examples>

<patterns>
- Use `os.Open` directly and check properties with `File.Stat()` to eliminate the TOCTOU gap
- Handle file operations inside `filepath.Walk` using the file descriptor directly instead of re-opening by path
- Replace `os.Exists` + open patterns with direct `os.Open` — handle `os.ErrNotExist` in the error path
</patterns>

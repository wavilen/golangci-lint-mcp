# staticcheck: SA1013

<instructions>
Detects `context.Value` calls where the return value is used without a type assertion. `context.Value` returns `interface{}` (or `any`), so accessing it without asserting the type defeats type safety and can cause nil pointer dereferences at runtime.

Always assert the type when retrieving values from context: `val, ok := ctx.Value(key).(string)`.
</instructions>

<examples>
## Bad
```go
func handler(ctx context.Context) {
    id := ctx.Value(requestIDKey) // id is interface{}, not string
    slog.Info("id", "value", id)
}
```

## Good
```go
func handler(ctx context.Context) {
    id, ok := ctx.Value(requestIDKey).(string)
    if !ok {
        id = "unknown"
    }
    slog.Info("id", "value", id)
}
```
</examples>

<patterns>
- Assigning `context.Value` result directly to a typed variable without assertion
- Using `context.Value` without checking the `ok` boolean from the type assertion
- Assuming the key always exists and has the expected type
</patterns>

</related>

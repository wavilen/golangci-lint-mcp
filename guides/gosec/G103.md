# G103

<instructions>
Audits the use of `unsafe` package operations (`unsafe.Pointer`, `unsafe.Slice`, `unsafe.String`, `unsafe.Sizeof` in pointer arithmetic). The `unsafe` package bypasses Go's type safety and memory guarantees, which can lead to memory corruption, data races, and undefined behavior.

Avoid `unsafe` operations unless absolutely necessary for performance-critical code or Cgo interop. When unavoidable, document the safety invariants clearly.
</instructions>

<examples>
## Bad
```go
import "unsafe"

func bytesToString(b []byte) string {
    return *(*string)(unsafe.Pointer(&b))
}
```

## Good
```go
func bytesToString(b []byte) string {
    return string(b)
}
```
</examples>

<patterns>
- `unsafe.Pointer` used to cast between incompatible types
- Pointer arithmetic via `unsafe.Add` or manual offset calculation
- `unsafe.Slice` used to reinterpret memory without bounds checking
- Casting between `[]byte` and `string` to avoid allocation (no longer needed in modern Go)
</patterns>

# G507

<instructions>
Detects imports of `golang.org/x/crypto/ripemd160`. RIPEMD-160 is a legacy hash function that, while not broken like MD5 or SHA-1, provides only 160 bits of output and lacks widespread adoption or modern security analysis. It does not offer meaningful advantages over SHA-256.

Use `crypto/sha256` or `crypto/sha512` for hashing. If RIPEMD-160 is required for a specific protocol (e.g., Bitcoin address generation), document that the usage is intentional and protocol-mandated.
</instructions>

<examples>
## Good
```go
import "crypto/sha256"

func hash(data []byte) []byte {
    h := sha256.Sum256(data)
    return h[:]
}
```
</examples>

<patterns>
- Remove `golang.org/x/crypto/ripemd160` imports — use `crypto/sha256` instead
- Ensure RIPEMD-160 usage is protocol-mandated only — document the specific protocol requirement
- Replace RIPEMD-160 with SHA-256 unless required by a specific protocol — RIPEMD-160 offers no security advantage
- Simplify dual-hashing schemes to SHA-256 alone — combining hashes does not meaningfully improve security
</patterns>

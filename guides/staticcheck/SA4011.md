# staticcheck: SA4011

<instructions>
Detects `break` statements inside `switch` or `select` within a `for` loop where the intent is likely to break out of the outer loop. A bare `break` inside `switch`/`select` only exits the switch/select block, not the enclosing loop. The loop continues to the next iteration.

Use a labeled `break` statement to exit the outer loop: `break LoopLabel`.
</instructions>

<examples>
## Bad
```go
for _, item := range items {
    switch item.Kind {
    case "stop":
        break // only breaks switch, loop continues
    }
}
```

## Good
```go
Loop:
    for _, item := range items {
        switch item.Kind {
        case "stop":
            break Loop // breaks the for loop
        }
    }
```
</examples>

<patterns>
- Use labeled `break` to exit a `for` loop from inside a `switch`
- Use labeled `break` to exit a `for` loop from inside a `select`
- Replace bare `break` with `break LoopLabel` when both switch and loop should exit
- Add loop labels for command parser and state machine patterns with nested switch
</patterns>

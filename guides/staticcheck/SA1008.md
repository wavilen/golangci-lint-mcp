# staticcheck: SA1008

<instructions>
Detects HTML entities (e.g., `&amp;`, `&lt;`) in `html/template` literal content where they are unnecessary. The `html/template` package already auto-escapes values during execution. Writing HTML entities in literal template text results in double-encoding — e.g., `&amp;amp;` renders as `&amp;`.

Use the raw characters in template literals and let the template engine handle escaping of dynamic values.
</instructions>

<examples>
## Good
```go
t := template.Must(template.New("x").Parse(`<p>Hello {{.Name}}</p>`))
// The engine auto-escapes .Name; static HTML can use literal characters
```
</examples>

<patterns>
- Remove manual HTML entities in `html/template` literals — the engine auto-escapes dynamic values
- Distinguish template auto-escaping from static content
- Replace browser-rendered HTML copy-pasted into template literals with raw characters
</patterns>

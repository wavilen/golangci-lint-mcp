# staticcheck: SA4024

<instructions>
Detects comparisons of `time.Time` values using `==` or `!=`. The `time.Time` type contains a `loc` (location) field and monotonic clock reading, so `==` comparison may return false for times that represent the same instant but differ in location or monotonic reading.

Use `t1.Equal(t2)` to compare time values for the same instant.
</instructions>

<examples>
## Good
```go
if startTime.Equal(endTime) {
    slog.Info("same time")
}
```
</examples>

<patterns>
- Replace `==` and `!=` on `time.Time` with `t1.Equal(t2)`
- Use `Equal()` to compare time values — struct equality ignores location/monotonic differences
- Use `Equal()` when comparing times from different timezones
- Use `Equal()` instead of `==` in tests where monotonic readings may differ
</patterns>

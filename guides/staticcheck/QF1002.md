# staticcheck: QF1002

<instructions>
Detects tagged switch statements with boolean expressions that could be written as untagged switch statements. A tagged switch like `switch true { case x: ... case y: ... }` should be an untagged switch `switch { case x: ... case y: ... }`.

Remove the `true` tag from the switch statement.
</instructions>

<examples>
## Good
```go
switch {
case x > 0:
    slog.Info("positive")
case x < 0:
    slog.Info("negative")
}
```
</examples>

<patterns>
- Replace `switch true { ... }` with `switch { ... }`
- Remove the boolean tag from switch with boolean case expressions
- Avoid `switch` on a constant boolean — use `if` instead
- Simplify over-structured switch for simple boolean cases
</patterns>

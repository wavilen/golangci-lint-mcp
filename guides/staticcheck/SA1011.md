# staticcheck: SA1011

<instructions>
Detects misuse of `encoding/xml.CharData`. `CharData` represents character data in XML. A common mistake is comparing `CharData` to a string directly — `CharData` is a `[]byte` alias and must be converted or used with its methods.
</instructions>

<examples>
## Bad
```go
var cd xml.CharData = []byte("hello")
if cd == "hello" { // type mismatch: CharData vs string
    slog.Info("match")
}
```

## Good
```go
var cd xml.CharData = []byte("hello")
if string(cd) == "hello" {
    slog.Info("match")
}
```
</examples>

<patterns>
- Comparing `xml.CharData` directly with a string literal
- Using `xml.CharData` where `[]byte` or `string` is expected without conversion
- Ignoring the `Copy()` requirement when retaining `CharData` beyond the callback scope
</patterns>

# staticcheck: S1027

<instructions>
Detects explicit `return` statements at the end of functions that return no values. In Go, functions naturally return at the end of their body — a bare `return` is unnecessary.

Remove the redundant return statement.
</instructions>

<examples>
## Bad
```go
func sayHi() {
    slog.Info("hi")
    return
}
```

## Good
```go
func sayHi() {
    slog.Info("hi")
}
```
</examples>

<patterns>
- Bare return at the end of a void function
- return statement with no value at the end of a function
- Redundant returns added by habit from other languages
- return as the last statement in a function body
</patterns>

# G404

<instructions>
Detects use of `math/rand` instead of `crypto/rand` for security-sensitive operations. `math/rand` uses a deterministic pseudo-random number generator that can be predicted, making generated tokens, keys, or nonces vulnerable.

Use `crypto/rand` for any value that must be unpredictable (tokens, keys, nonces, salts). Only use `math/rand` for non-security purposes like simulations or shuffling.
</instructions>

<examples>
## Good
```go
import "crypto/rand"

token := make([]byte, 32)
_, err := rand.Read(token)
if err != nil {
    return err
}
```
</examples>

<patterns>
- Use `crypto/rand` for session tokens, API keys, and passwords — `math/rand` is not cryptographically secure
- Replace `math/rand` with `crypto/rand` for nonces and IVs — use `io.ReadFull(rand.Reader, buf)`
- Remove explicit seeding of `math/rand` for security purposes — use `crypto/rand` instead
</patterns>

# staticcheck: S1001

<instructions>
Detects `for` loops that copy elements from one slice to another element-by-element. The built-in `copy()` function is more concise, clearer in intent, and often faster since it can use `memmove` internally.

Replace the loop with `copy(dst, src)`.
</instructions>

<examples>
## Bad
```go
src := []int{1, 2, 3}
dst := make([]int, len(src))
for i, v := range src {
    dst[i] = v
}
```

## Good
```go
src := []int{1, 2, 3}
dst := make([]int, len(src))
copy(dst, src)
```
</examples>

<patterns>
- Replace manual element-by-element slice copying with `copy(dst, src)`
- Use `copy` instead of index assignment into a pre-allocated destination
- Replace loop-based `append` with `copy` when moving entire slices
- Use `copy` for transferring between slices of the same type
</patterns>

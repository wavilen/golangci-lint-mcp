# staticcheck: SA5003

<instructions>
Detects `defer` statements inside `if` bodies. While `defer` is legal inside any block, a `defer` in an `if` still defers until the *function* returns, not until the `if` block exits. This is often confusing and rarely the intended behavior.

Move the `defer` to the outer function scope, or use a separate function to limit the defer scope.
</instructions>

<examples>
## Bad
```go
func process(path string) error {
    if cond {
        f, err := os.Open(path)
        if err != nil {
            return err
        }
        defer f.Close() // deferred until process() returns, not until if exits
        // ... use f
    }
    // f is still open here
    return nil
}
```

## Good
```go
func process(path string) error {
    if cond {
        if err := processFile(path); err != nil {
            return err
        }
    }
    return nil
}

func processFile(path string) error {
    f, err := os.Open(path)
    if err != nil {
        return err
    }
    defer f.Close() // deferred until processFile() returns
    return nil
}
```
</examples>

<patterns>
- `defer` inside `if` blocks where early cleanup is intended
- Conditional defer of resource cleanup
- `defer mu.Unlock()` inside an `if` block
- Defer of logging or tracing calls inside conditionals
</patterns>

# staticcheck: SA6005

<instructions>
Detects inefficient string concatenation inside loops using `+=`. Each `+=` on a string allocates a new string because strings are immutable. This leads to O(n²) allocation behavior for n concatenations.

Use `strings.Builder` to efficiently build strings in a loop with O(n) allocation.
</instructions>

<examples>
## Bad
```go
result := ""
for _, s := range parts {
    result += s // O(n²) allocations
}
```

## Good
```go
var b strings.Builder
for _, s := range parts {
    b.WriteString(s)
}
result := b.String()
```
</examples>

<patterns>
- Replace string `+=` in loops with `strings.Builder` for O(n) allocation
- Use `strings.Builder` for log message construction instead of repeated concatenation
- Use `strings.Builder` for building CSV or delimited strings iteratively
- Replace query-building concatenation with `strings.Builder`
</patterns>

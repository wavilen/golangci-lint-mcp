# staticcheck: S1023

<instructions>
Detects hardcoded paths like `"/dev/null"` or `"NUL"` instead of using `os.DevNull`. The `os.DevNull` constant is portable across operating systems and makes the intent explicit.

Replace the hardcoded device path with `os.DevNull`.
</instructions>

<examples>
## Bad
```go
f, err := os.Open("/dev/null")
```

## Good
```go
f, err := os.Open(os.DevNull)
```
</examples>

<patterns>
- Using "/dev/null" on Unix-specific code when os.DevNull is portable
- Using "NUL" on Windows-specific code
- Hardcoding the null device path in test fixtures
- Discarding output by writing to hardcoded /dev/null paths
</patterns>

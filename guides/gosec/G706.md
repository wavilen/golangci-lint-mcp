# G706

<instructions>
Detects log injection where tainted user input flows into log output without sanitization. Attackers can inject fake log entries with newline characters to forge audit trails, hide malicious activity, or confuse log analysis tools.

Sanitize user input before logging by stripping or encoding newline characters (`\n`, `\r`). Use structured logging (e.g., `slog` with key-value pairs) which naturally separates user data from log framing.
</instructions>

<examples>
## Good
```go
import "log/slog"

func handleLogin(username string) {
    // Structured logging separates user data from log format.
    slog.Info("user logged in", "username", username)
}

// Alternative: sanitize inline.
func sanitizeForLog(s string) string {
    s = strings.ReplaceAll(s, "\n", "\\n")
    s = strings.ReplaceAll(s, "\r", "\\r")
    return s
}
```
</examples>

<patterns>
- Sanitize user input before logging — strip newline characters or use structured logging
- Remove or encode newline characters in user input before writing to log output
- Replace multi-line user input with single-line representations before logging
- Use structured logging with tamper-evident formats for audit trails — never embed raw user input
</patterns>

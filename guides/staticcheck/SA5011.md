# staticcheck: SA5011

<instructions>
Detects possible nil pointer dereferences where a pointer is checked for nil in one branch but dereferenced in another branch without the nil check. This leads to a panic at runtime when the pointer is nil.

Add a nil check before dereferencing the pointer, or restructure the code to ensure the pointer is always non-nil in the dereferencing path.
</instructions>

<examples>
## Bad
```go
func getLen(s *string) int {
    if s == nil {
        return 0
    }
    slog.Info("value", "s", *s)
    return len(*s)
    // if another path reaches here without nil check, *s panics
}
```

## Good
```go
func getLen(s *string) int {
    if s == nil {
        return 0
    }
    return len(*s) // s is guaranteed non-nil here
}
```
</examples>

<patterns>
- Pointer dereference after a nil check in a different code path
- Accessing struct fields through a nil pointer
- Method calls on nil receivers that don't handle nil
- Returning pointer from function, caller doesn't check nil
</patterns>

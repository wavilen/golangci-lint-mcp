# G201

<instructions>
Detects SQL query construction using `fmt.Sprintf` or `fmt.Sprintf`-style string formatting. Building SQL queries with string formatting allows SQL injection when user input is embedded directly into the query string.

Use parameterized queries with `database/sql` placeholders (`$1`, `?`, or `:param`). Never interpolate user input into SQL strings.
</instructions>

<examples>
## Bad
```go
query := fmt.Sprintf("SELECT * FROM users WHERE id = %s", userID)
rows, _ := db.Query(query)
```

## Good
```go
rows, err := db.Query("SELECT * FROM users WHERE id = $1", userID)
if err != nil {
    return err
}
```
</examples>

<patterns>
- `fmt.Sprintf` used to build SQL query strings with user input
- `string + string` concatenation in SQL WHERE clauses
- `fmt.Fprintf` writing SQL to a buffer with dynamic values
</patterns>

# staticcheck: S1009

<instructions>
Detects redundant nil checks before operations that already handle nil safely. In Go, `range` over a nil slice is a no-op, `len(nil)` returns 0, and type assertions on nil interfaces panic but `x.(*T)` where x is already checked is redundant.

Remove the unnecessary nil guard.
</instructions>

<examples>
## Bad
```go
if items != nil {
    for _, v := range items {
        slog.Info("value", "v", v)
    }
}
```

## Good
```go
for _, v := range items {
    slog.Info("value", "v", v)
}
```
</examples>

<patterns>
- Checking nil before range (nil slice range is safe — no iterations)
- Checking nil before len() (len(nil) returns 0)
- Checking nil before a type assertion that is already guarded
- Double nil checks where the inner block already handles nil
</patterns>

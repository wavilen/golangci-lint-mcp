# staticcheck: SA4032

<instructions>
Detects `io.Copy` calls where the destination is `os.Stdout` or `os.Stderr`. These file objects implement `io.Writer` directly, so `io.Copy` works but the more specific `os.File` methods may be more appropriate. More importantly, this check often catches cases where a buffered writer was intended but the raw `os.File` is used directly.

Consider using `io.CopyBuffer` with a custom buffer size, or wrapping `os.Stdout` in a `bufio.Writer` for better performance.
</instructions>

<examples>
## Bad
```go
io.Copy(os.Stdout, resp.Body) // unbuffered copy to stdout
```

## Good
```go
_, err := io.Copy(os.Stdout, resp.Body)
if err != nil {
    log.Fatal(err)
}
```
</examples>

<patterns>
- `io.Copy(os.Stdout, reader)` for direct console output
- `io.Copy(os.Stderr, reader)` for error output
- Using `io.Copy` when `ReadFrom` or buffered writing would be better
- Copying to files without checking the error return
</patterns>

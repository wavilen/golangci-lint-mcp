# staticcheck: ST1011

<instructions>
Detects inconsistent variable naming within a function or package. This includes using different naming styles for similar variables (e.g., mixing `userID` and `user_id`), or using abbreviations inconsistently.

Use consistent naming conventions throughout the codebase.
</instructions>

<examples>
## Bad
```go
func process(userId int, userName string) {
    user_id := getUserId(userId)
}
```

## Good
```go
func process(userID int, userName string) {
    id := getUserID(userID)
}
```
</examples>

<patterns>
- Standardize variable naming to camelCase
- Use consistent abbreviations
- Keep naming consistent across functions in the same package
- Avoid variable names that shadow imported package names
</patterns>

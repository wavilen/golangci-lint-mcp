# staticcheck: SA1014

<instructions>
Detects calls to `encoding/json.Marshal` or `Unmarshal` with a map that has a non-string key, or other types that JSON cannot represent. JSON object keys must be strings — maps with integer, float, or struct keys will either fail to encode or produce unexpected output.
</instructions>

<examples>
## Bad
```go
m := map[int]string{1: "one", 2: "two"}
data, _ := json.Marshal(m) // produces {"1":"one","2":"two"} — keys become strings
```

## Good
```go
type Item struct {
    ID    int    `json:"id"`
    Value string `json:"value"`
}
items := []Item{{1, "one"}, {2, "two"}}
data, _ := json.Marshal(items)
```
</examples>

<patterns>
- Replace maps with non-string keys with `[]struct` before `json.Marshal`
- Use structs with JSON tags instead of expecting numeric keys in JSON output
- Avoid relying on map key ordering in marshaled JSON — JSON objects are unordered
</patterns>

# staticcheck: S1011

<instructions>
Detects `for` loops that append elements one at a time to a slice. Using a single `append` call with all elements (or variadic expansion) is more efficient and readable — it minimizes allocations.

Replace the loop with a single `append` call.
</instructions>

<examples>
## Bad
```go
for _, v := range newItems {
    items = append(items, v)
}
```

## Good
```go
items = append(items, newItems...)
```
</examples>

<patterns>
- Loop appending individual elements when append accepts variadic args
- For-range that only calls append in the body
- Building a slice one element at a time instead of using spread
- Appending from one slice to another element-by-element
</patterns>

# staticcheck: ST1001

<instructions>
Detects dot imports (`import . "pkg"`) which make the imported names appear in the current file's namespace without qualification. Dot imports hurt readability and maintainability because it's unclear where a name originates.

Use regular imports and qualify names with the package name.
</instructions>

<examples>
## Bad
```go
import . "fmt"

func main() {
    Println("hello")
}
```

## Good
```go
import "fmt"

func main() {
    fmt.Println("hello")
}
```
</examples>

<patterns>
- Replace dot imports with qualified imports in application code
- Avoid dot imports for test assertion functions
- Replace dot-imported test packages with qualified imports
- Remove dot imports used for domain-specific language embedding
</patterns>

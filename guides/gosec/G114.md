# G114

<instructions>
Detects use of `http.ListenAndServe`, `http.ListenAndServeTLS`, `http.Serve`, or `http.ServeTLS` which do not support setting timeouts. These convenience functions create an `http.Server` with no timeout configuration, leaving the server vulnerable to Slowloris and connection exhaustion attacks.

Create an explicit `http.Server` struct with `ReadHeaderTimeout`, `ReadTimeout`, `WriteTimeout`, and `IdleTimeout` set, then call `srv.ListenAndServe()`.
</instructions>

<examples>
## Good
```go
srv := &http.Server{
    Addr:              ":8080",
    Handler:           handler,
    ReadHeaderTimeout: 5 * time.Second,
    ReadTimeout:       10 * time.Second,
    WriteTimeout:      30 * time.Second,
    IdleTimeout:       120 * time.Second,
}
srv.ListenAndServe()
```
</examples>

<patterns>
- Replace `http.ListenAndServe` with a custom `http.Server` that sets `ReadTimeout` and `WriteTimeout`
- Configure timeouts on `http.Server` before calling `ListenAndServeTLS` — set at minimum `ReadHeaderTimeout`
- Wrap `net.Listener` with timeout configuration or use `http.Server` with explicit timeout fields
</patterns>

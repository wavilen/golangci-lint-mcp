# staticcheck: S1005

<instructions>
Detects unnecessary blank identifier assignments. This includes receiving from a closed channel into a blank identifier, or assigning to `_` when the value is never used. These patterns add noise without benefit.

Remove the unnecessary blank assignment or restructure the code.
</instructions>

<examples>
## Bad
```go
_ = <-ch  // receiving only for side effect
_ = compute()
```

## Good
```go
<-ch      // receive and discard directly
compute() // just call the function
```
</examples>

<patterns>
- Assigning function return values to blank identifier when not needed
- Receiving from a channel into blank identifier
- Discarding map lookup boolean with `_` before checking it
- Using `_ =` before a call where the side effect is the only purpose
</patterns>

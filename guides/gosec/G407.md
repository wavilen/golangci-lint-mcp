# G407

<instructions>
Detects hardcoded initialization vectors (IV) or nonces used for encryption. Reusing an IV/nonce with the same key in AES-GCM or ChaCha20-Poly1305 completely breaks confidentiality and authenticity — an attacker can recover the key and forge messages.

Generate a unique, random IV/nonce for each encryption operation using `crypto/rand`. Prepend the nonce to the ciphertext for storage/transmission.
</instructions>

<examples>
## Good
```go
nonce := make([]byte, gcm.NonceSize())
if _, err := rand.Read(nonce); err != nil {
    return err
}
ciphertext := gcm.Seal(nonce, nonce, plaintext, nil) // nonce prepended to ciphertext
```
</examples>

<patterns>
- Use `crypto/rand` to generate unique nonces for each encryption — never hardcode AES-GCM nonces
- Use `crypto/rand` to generate a random IV for each CBC encryption call — never reuse IVs across messages
- Use `crypto/rand` to generate nonces and IVs — avoid deriving them from counters or timestamps
</patterns>

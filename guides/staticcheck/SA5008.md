# staticcheck: SA5008

<instructions>
Detects struct fields where tags for a given key use inconsistent formats or naming conventions. For example, some fields using `json:"fieldName"` while others use `json:"field_name"` or different casing styles. Inconsistency makes serialization unpredictable.

Standardize all struct tags for each key to use the same naming convention (e.g., all snake_case or all camelCase).
</instructions>

<examples>
## Bad
```go
type User struct {
    FirstName string `json:"first_name"`
    LastName  string `json:"lastName"` // inconsistent: snake_case vs camelCase
}
```

## Good
```go
type User struct {
    FirstName string `json:"first_name"`
    LastName  string `json:"last_name"`
}
```
</examples>

<patterns>
- Mixed `snake_case` and `camelCase` in JSON tags
- Some fields with `db` tags and others without
- Inconsistent `omitempty` usage across similar fields
- Tags added at different times following different conventions
</patterns>

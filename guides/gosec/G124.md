# G124

<instructions>
Detects HTTP cookies set without security attributes (`Secure`, `HttpOnly`, `SameSite`). Missing these flags allows cookies to be stolen via XSS (`HttpOnly`), sent over plaintext connections (`Secure`), or attached to cross-site requests (`SameSite`).

Always set `Secure: true`, `HttpOnly: true`, and `SameSite: http.SameSiteLaxMode` (or `SameSiteStrictMode`) on session cookies.
</instructions>

<examples>
## Bad
```go
http.SetCookie(w, &http.Cookie{
    Name:  "session",
    Value: token,
})
```

## Good
```go
http.SetCookie(w, &http.Cookie{
    Name:     "session",
    Value:    token,
    Secure:   true,
    HttpOnly: true,
    SameSite: http.SameSiteLaxMode,
    Path:     "/",
    MaxAge:   3600,
})
```
</examples>

<patterns>
- Cookies missing the `Secure` flag, allowing transmission over HTTP
- Session cookies without `HttpOnly`, accessible to JavaScript (XSS risk)
- Missing `SameSite` attribute, enabling CSRF attacks via cross-site requests
</patterns>

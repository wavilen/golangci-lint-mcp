# staticcheck: SA4010

<instructions>
Detects `append` calls whose result is not assigned back to a variable. Since `append` may allocate a new backing array, discarding the result means the original slice is unchanged and the appended elements are lost. This is always a bug.

Assign the result of `append` back to the same variable: `s = append(s, x)`.
</instructions>

<examples>
## Bad
```go
items := []string{"a", "b"}
append(items, "c") // result discarded, items unchanged
```

## Good
```go
items := []string{"a", "b"}
items = append(items, "c")
```
</examples>

<patterns>
- Assign the result of `append` back to the slice variable — `s = append(s, x)`
- Reassign map slice entries after append: `m[key] = append(m[key], x)`
- Capture `append` results in chained operations to avoid losing them
- Remember `append` returns a new slice — it does not mutate in-place
</patterns>

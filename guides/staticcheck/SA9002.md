# staticcheck: SA9002

<instructions>
Detects `os.FileMode` or `syscall.Permissions` values written as non-octal integer literals. File permissions like `0644` must be written in octal notation (leading `0`) to be readable and correct. Using decimal notation like `644` gives a completely different permission (octal 1204).

Use octal notation with leading `0` for file permissions: `0644`, `0755`, `0600`.
</instructions>

<examples>
## Bad
```go
os.Chmod("file.txt", 644) // decimal 644 = octal 1204, wrong permissions
```

## Good
```go
os.Chmod("file.txt", 0644) // octal: owner rw, group r, others r
```
</examples>

<patterns>
- Use octal notation `0644` instead of decimal `644` for file permissions
- Use octal literals for permission bits — not decimal integers
- Use octal notation in `os.Mkdir` permission arguments
- Use octal notation in `os.WriteFile` mode arguments
</patterns>

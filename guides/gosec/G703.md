# G703

<instructions>
Detects path traversal where tainted user input flows into file system operations, allowing attackers to access files outside the intended directory. By injecting `../` sequences or absolute paths, an attacker can read or write arbitrary files on the system.

Sanitize and validate all user-supplied paths. Resolve the path against a trusted base directory and verify the result stays within that directory. Reject paths containing `..` components or absolute paths.
</instructions>

<examples>
## Bad
```go
func readFile(userInput string) ([]byte, error) {
    // Attacker input "../../../etc/passwd" escapes the data directory
    path := filepath.Join("./data", userInput)
    return os.ReadFile(path)
}
```

## Good
```go
func readFile(userInput string) ([]byte, error) {
    baseDir := "/app/data"
    path := filepath.Join(baseDir, userInput)
    // Resolve symlinks and verify the path is within baseDir
    absPath, err := filepath.Abs(path)
    if err != nil {
        return nil, err
    }
    if !strings.HasPrefix(absPath, baseDir+string(filepath.Separator)) {
        return nil, fmt.Errorf("path traversal detected")
    }
    return os.ReadFile(absPath)
}
```
</examples>

<patterns>
- User-supplied filenames passed directly to `os.Open`, `os.ReadFile`, or `filepath.Join`
- `../` sequences in URL path components used to construct file paths
- Absolute paths from user input overriding the intended base directory
- Symlinks created by attackers pointing outside the allowed directory
</patterns>

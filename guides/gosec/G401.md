# G401

<instructions>
Detects use of MD5 (`crypto/md5`) or SHA-1 (`crypto/sha1`) for cryptographic purposes. Both algorithms are broken: MD5 has known collision attacks, and SHA-1 is vulnerable to practical collision attacks. They are unsuitable for signatures, password hashing, or integrity verification.

Use SHA-256 (`crypto/sha256`) or SHA-512 for hashing. For password storage, use `bcrypt`, `scrypt`, or `argon2`.
</instructions>

<examples>
## Bad
```go
import "crypto/md5"

hash := md5.Sum([]byte(password))
```

## Good
```go
import "crypto/sha256"

hash := sha256.Sum256([]byte(data))
```
</examples>

<patterns>
- `crypto/md5` used for checksums, signatures, or password hashing
- `crypto/sha1` used where collision resistance is expected
- MD5 used for data integrity verification in security-sensitive contexts
</patterns>

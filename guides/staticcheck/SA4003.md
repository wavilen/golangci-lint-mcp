# staticcheck: SA4003

<instructions>
Detects comparisons of the form `x >= 0` or `x > 0` (with unsigned types, depending on context) where the comparison is always true due to the unsigned type. This is the complement of SA4001 — unsigned values are never negative, so `>= 0` is trivially true.

Remove the redundant comparison or use a signed integer type if negative values need to be tracked.
</instructions>

<examples>
## Bad
```go
var count uint
if count >= 0 {
    slog.Info("always true for unsigned")
}
```

## Good
```go
var count uint
if count > 0 {
    slog.Info("has items")
}
```
</examples>

<patterns>
- `uint >= 0` which is always true
- `uintptr >= 0` comparisons
- Guard checks on unsigned loop counters
- Redundant zero-bound checks on unsigned values
</patterns>

# G303

<instructions>
Detects temporary files created using predictable paths (e.g., hardcoded temp file names). Predictable paths allow attackers to pre-create symlinks or files at that location, leading to data corruption or privilege escalation via symlink attacks.

Use `os.CreateTemp` (or `os.MkdirTemp`) to generate unpredictable file/directory names. Never hardcode temp file paths.
</instructions>

<examples>
## Bad
```go
f, _ := os.Create("/tmp/myapp-tmp-file")
```

## Good
```go
f, err := os.CreateTemp("", "myapp-*.tmp")
if err != nil {
    return err
}
defer os.Remove(f.Name())
defer f.Close()
```
</examples>

<patterns>
- Hardcoded temp file paths like `/tmp/myapp-data`
- Predictable temp file names constructed from timestamps or process IDs
- Manual temp file creation without `os.CreateTemp`
</patterns>

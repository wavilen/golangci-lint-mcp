# staticcheck: SA4016

<instructions>
Detects implicit memory aliasing in `for range` loops where the address of the range variable is taken. Since the range variable is a copy, taking its address gives a pointer to a variable that is overwritten each iteration. All pointers point to the same memory with the last iteration's value.

Create a local copy inside the loop body and take the address of that copy instead.
</instructions>

<examples>
## Good
```go
var ptrs []*int
for _, v := range []int{1, 2, 3} {
    v := v // new variable per iteration
    ptrs = append(ptrs, &v)
}
```
</examples>

<patterns>
- Create a local copy `v := v` before taking `&v` in a range loop
- Store pointers to loop-local copies — not the range variable — in slices or maps
- Pass the address of a loop-local copy to goroutines, not the range variable
- Build pointer collections using per-iteration copies during iteration
</patterns>

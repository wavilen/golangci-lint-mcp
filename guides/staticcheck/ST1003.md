# staticcheck: ST1003

<instructions>
Detects constants that use CamelCase or mixedCase naming instead of the Go convention of UPPER_CASE or Go-style PascalCase for exported constants. While Go doesn't enforce a strict convention, staticcheck flags inconsistencies where a constant's naming doesn't follow idiomatic patterns.

Use consistent naming: UPPER_CASE for acronyms or Go's standard PascalCase for exported constants.
</instructions>

<examples>
## Good
```go
const MaxRetries = 3
const HTTPPort = 8080
```
</examples>

<patterns>
- Export constants that should be accessible outside the package
- Use PascalCase or UPPER_CASE for constants instead of camelCase
- Standardize naming conventions within the same const block
- Use consistent casing for abbreviations
</patterns>

# staticcheck: SA9007

<instructions>
Detects `delete` calls on map keys that may not exist. While `delete` on a non-existent key is safe (no-op), it often indicates a logic error where the code assumes the key exists. If the key should exist, its absence may signal a bug in the program's logic.

Check if the key exists before deleting, or use the comma-ok idiom if the deletion is conditional. If the no-op behavior is intentional, a comment should clarify that.
</instructions>

<examples>
## Bad
```go
delete(cache, key) // is key actually in the cache?
```

## Good
```go
if _, ok := cache[key]; ok {
    delete(cache, key)
}
```
</examples>

<patterns>
- Deleting map keys without verifying existence first
- Cache invalidation where missing key may indicate a bug
- Cleanup maps where absence of key is unexpected
- Map operations assuming a key was previously set
</patterns>

# G408

<instructions>
Detects stateful misuse of `ssh.PublicKeyCallback` that can lead to authentication bypass. When `PublicKeyCallback` stores verification state (e.g., caching the accepted key) and is reused across multiple connections or auth attempts, an attacker who successfully authenticates once may bypass future checks.

Make `PublicKeyCallback` stateless. Each invocation should independently verify the presented public key against the authorized keys. Do not cache successful verification results across calls.
</instructions>

<examples>
## Good
```go
config := &ssh.ServerConfig{
    PublicKeyCallback: func(conn ssh.ConnMetadata, key ssh.PublicKey) (*ssh.Permissions, error) {
        // Stateless: each call verifies independently
        if !isAuthorizedKey(key) {
            return nil, fmt.Errorf("unauthorized key")
        }
        return &ssh.Permissions{}, nil
    },
}
```
</examples>

<patterns>
- Remove cached SSH key verification state — verify keys per-connection instead of caching results
- Ensure `PublicKeyCallback` verifies the key on every call — never skip based on prior results
- Ensure each connection gets a fresh `ssh.ServerConfig` or that callbacks have no mutable shared state
</patterns>

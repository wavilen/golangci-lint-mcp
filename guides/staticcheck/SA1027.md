# staticcheck: SA1027

<instructions>
Detects `atomic.Value.Store` calls where the concrete type differs from the type of the first value stored. `atomic.Value` requires all stored values to have the same concrete type — the first `Store` or `Load` sets the type permanently. Mixing types causes a panic at runtime.
</instructions>

<examples>
## Good
```go
var v atomic.Value
v.Store("hello")
v.Store("world") // same concrete type: string
```
</examples>

<patterns>
- Ensure all `atomic.Value.Store` calls use the same concrete type
- Handle the initial `Load` returning nil before the first `Store`
- Replace `atomic.Value` with type-safe alternatives when storing heterogeneous data
</patterns>

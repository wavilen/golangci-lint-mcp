# staticcheck: SA4000

<instructions>
Detects identical expressions on both sides of a binary operator, such as `x == x` or `x - x`. These comparisons are always true or always false and indicate a typo or copy-paste error where a different operand was intended.

Replace one side with the intended expression, or remove the redundant comparison entirely.
</instructions>

<examples>
## Bad
```go
if x == x {
    slog.Info("always true")
}
```

## Good
```go
if x == y {
    slog.Info("compares two values")
}
```
</examples>

<patterns>
- Replace self-comparisons (`a == a`, `a != a`) with comparisons against the intended different variable
- Remove subtracting or XORing a value with itself (`x - x`, `x ^ x`) — the result is always zero
- Simplify logical AND/OR with the same expression (`a && a`) to the expression itself
- Fix copy-paste errors where one operand was meant to be different
</patterns>

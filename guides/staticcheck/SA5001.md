# staticcheck: SA5001

<instructions>
Detects `defer file.Close()` or similar patterns where the error return from `Close` is not checked. For writable files and network connections, `Close` can fail — typically because buffered data could not be flushed. Ignoring this error can silently lose data.

Check the error from `Close` in the defer. Use a named return variable to propagate the error.
</instructions>

<examples>
## Bad
```go
f, err := os.Create("output.txt")
if err != nil {
    return err
}
defer f.Close() // error from Close ignored
```

## Good
```go
f, err := os.Create("output.txt")
if err != nil {
    return err
}
defer func() {
    if cerr := f.Close(); cerr != nil && err == nil {
        err = cerr
    }
}()
```
</examples>

<patterns>
- `defer f.Close()` for writable files without error check
- `defer resp.Body.Close()` where response body error matters
- `defer conn.Close()` for network connections
- Logging files where flush-on-close failures should be reported
</patterns>

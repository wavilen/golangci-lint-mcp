# staticcheck: ST1004

<instructions>
Detects uses of `fmt.Errorf` or `errors.New` where the error message starts with a capital letter. By Go convention, error messages should not start with a capital letter because they may appear mid-sentence when wrapped.

Lowercase the first letter of the error message string.
</instructions>

<examples>
## Bad
```go
return fmt.Errorf("Invalid input: %d", n)
```

## Good
```go
return fmt.Errorf("invalid input: %d", n)
```
</examples>

<patterns>
- Error messages starting with uppercase letters
- Capitalized error strings in errors.New calls
- Capitalized format strings in fmt.Errorf
- User-facing error messages that look like standalone sentences
</patterns>

<related>
ST1005, ST1000, S1028

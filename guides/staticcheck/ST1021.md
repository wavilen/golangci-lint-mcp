# staticcheck: ST1021

<instructions>
Detects unnecessary code patterns in variable and type declarations. This includes using `var` where `const` would be appropriate, or declaring variables that could be inlined.

Use `const` for values that never change, or inline the variable.
</instructions>

<examples>
## Bad
```go
var defaultTimeout = 30 * time.Second
```

## Good
```go
const defaultTimeout = 30 * time.Second
```
</examples>

<patterns>
- Using var for values that should be const
- Declaring variables at package level that never change
- Using var for configuration constants
- Mutable variables that are never reassigned
</patterns>

<related>
ST1003, S1021, S1036

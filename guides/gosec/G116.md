# G116

<instructions>
Detects Trojan Source attacks that use bidirectional Unicode control characters (such as U+202A-U+202E, U+2066-U+2069) to visually reorder code in misleading ways. An attacker can embed these invisible characters so code review shows one logic but the compiler sees different logic.

Reject source files containing bidirectional Unicode control characters. Use editor settings that detect or strip these characters, and add pre-commit hooks to scan for them.
</instructions>

<examples>
## Bad
```go
// Invisible U+202E (RIGHT-TO-LEFT OVERRIDE) embedded in identifier
accessLevel := "user\u202Eadmin" // visually appears as "accessLevel = "user"
if accessLevel != "admin" {
    return
}
```

## Good
```go
// No Unicode control characters — identifiers are plain ASCII
accessLevel := "user"
if accessLevel != "admin" {
    return
}
```
</examples>

<patterns>
- Remove bidirectional override characters (U+202A-U+202E) from string literals and identifiers
- Remove invisible directional markers (U+2066-U+2069) from source code — use Unicode scanners if needed
- Check contributions for hidden Unicode control characters before merging — use editor/CI linting
</patterns>

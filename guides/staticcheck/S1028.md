# staticcheck: S1028

<instructions>
Detects overly complex error construction patterns. This includes using `fmt.Errorf` with no format verbs (should be `errors.New`), or wrapping errors with `fmt.Errorf("%s", err)` instead of `errors.Wrap(err, "context")` from `github.com/pkg/errors`.

Use `errors.New` for static messages and `errors.Wrap` from `github.com/pkg/errors` for wrapping.
</instructions>

<examples>
## Bad
```go
err := fmt.Errorf("something went wrong") // no format verbs
```

## Good
```go
err := errors.New("something went wrong")
```
</examples>

<patterns>
- Using fmt.Errorf without format verbs — use errors.New instead
- Using fmt.Errorf("%s", err) — lose the error chain
- Constructing errors with Sprintf then errors.New — just use fmt.Errorf
- Wrapping errors without %w so errors.Is/As won't work
</patterns>

# staticcheck: S1010

<instructions>
Detects slice expressions of the form `a[low:high]` where the high bound equals `len(a)`. Since omitting the high bound defaults to the slice length, `a[low:]` is equivalent to `a[low:len(a)]` but more concise.

Remove the redundant `len(a)` from the slice expression.
</instructions>

<examples>
## Bad
```go
suffix := data[2:len(data)]
```

## Good
```go
suffix := data[2:]
```
</examples>

<patterns>
- Using s[i:len(s)] where s[i:] is equivalent
- Three-index slicing with redundant length cap
- Re-slicing a slice with explicit length when omitted defaults correctly
- Copy-paste from code that originally used a different upper bound
</patterns>

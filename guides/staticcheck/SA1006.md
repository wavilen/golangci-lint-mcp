# staticcheck: SA1006

<instructions>
Detects `fmt.Printf` or `fmt.Sprintf` calls where an `*http.Request` is passed as an argument. The `Request` type has a large `String()` output that leaks headers, cookies, and query parameters. This is an information disclosure risk in logs and error messages.

Log only the specific fields you need from the request (URL, method, or request ID) instead of the entire object.
</instructions>

<examples>
## Bad
```go
func handler(w http.ResponseWriter, r *http.Request) {
    log.Printf("received request: %v", r) // leaks headers, cookies, auth tokens
}
```

## Good
```go
func handler(w http.ResponseWriter, r *http.Request) {
    log.Printf("received %s %s", r.Method, r.URL.Path)
}
```
</examples>

<patterns>
- Passing `*http.Request` directly to `fmt.Printf`, `log.Printf`, or `fmt.Sprintf`
- Including the full request in error messages that may be returned to clients
- Logging requests for debugging without filtering sensitive fields
</patterns>

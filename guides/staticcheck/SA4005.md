# staticcheck: SA4005

<instructions>
Detects ineffective assignments to struct fields inside for loops where the assigned value is immediately overwritten in the next iteration or never used. The assignment has no observable effect because the field is reassigned before it is read.

Restructure the code so the assignment happens only when the value is actually used, or move the assignment outside the loop.
</instructions>

<examples>
## Bad
```go
for i := range items {
    result.Field = items[i].Value // overwritten each iteration
}
return result // only last assignment matters
```

## Good
```go
for i := range items {
    process(items[i])
}
result.Field = items[len(items)-1].Value // single assignment if only last matters
```
</examples>

<patterns>
- Struct field assigned in every loop iteration but only last value used
- Maps populated in a loop where keys are overwritten
- Variable assignments inside loops where the value is never read
- Accumulator variables that are reset each iteration
</patterns>

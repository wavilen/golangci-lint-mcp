# staticcheck: ST1003

<instructions>
Detects constants that use CamelCase or mixedCase naming instead of the Go convention of UPPER_CASE or Go-style PascalCase for exported constants. While Go doesn't enforce a strict convention, staticcheck flags inconsistencies where a constant's naming doesn't follow idiomatic patterns.

Use consistent naming: UPPER_CASE for acronyms or Go's standard PascalCase for exported constants.
</instructions>

<examples>
## Bad
```go
const maxRetries = 3
const httpPort = 8080
```

## Good
```go
const MaxRetries = 3
const HTTPPort = 8080
```
</examples>

<patterns>
- Unexported constants that should be exported by convention
- Constants using camelCase where PascalCase or UPPER_CASE is expected
- Mixed naming conventions within the same const block
- Abbreviations not using consistent casing (httpPort vs HTTPPort)
</patterns>

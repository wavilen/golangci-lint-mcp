# staticcheck: SA6001

<instructions>
Detects map indexing in expressions that should use the comma-ok idiom. When accessing a map with `m[key]`, the zero value is returned if the key doesn't exist. Using this result without checking can lead to silent bugs where missing keys are treated as valid zero values.

Use the two-value form `val, ok := m[key]` to distinguish between a missing key and a legitimate zero value.
</instructions>

<examples>
## Bad
```go
count := wordFreq[word]
count++ // increments zero even if key doesn't exist
wordFreq[word] = count
```

## Good
```go
if count, ok := wordFreq[word]; ok {
    wordFreq[word] = count + 1
} else {
    wordFreq[word] = 1
}
```
</examples>

<patterns>
- Map access without checking existence when zero value is meaningful
- Incrementing map values without checking if key exists
- Using map values as counters where absence should be handled differently
- Config maps where missing key should trigger an error, not default
</patterns>

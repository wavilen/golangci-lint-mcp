# G305

<instructions>
Detects file path traversal when extracting zip or tar archives. If archive entries contain paths with `../` sequences, extracting them can write files outside the target directory, overwriting sensitive files.

Sanitize each archive entry's path before extraction. Reject entries with `..` components or absolute paths. Verify the cleaned path remains within the target directory.
</instructions>

<examples>
## Bad
```go
reader, _ := zip.Openarchive(uploadedZip)
for _, f := range reader.File {
    rc, _ := f.Open()
    out, _ := os.Create(filepath.Join("/var/data", f.Name)) // f.Name may contain ../
    io.Copy(out, rc)
}
```

## Good
```go
reader, _ := zip.OpenReader(uploadedZip)
for _, f := range reader.File {
    clean := filepath.Clean(f.Name)
    if strings.Contains(clean, "..") || filepath.IsAbs(clean) {
        continue // skip malicious entries
    }
    target := filepath.Join("/var/data", clean)
    if !strings.HasPrefix(filepath.Clean(target), "/var/data/") {
        continue
    }
    rc, _ := f.Open()
    out, _ := os.Create(target)
    io.Copy(out, rc)
}
```
</examples>

<patterns>
- Sanitize zip entry names before extraction — reject paths containing `..` or absolute paths
- Validate tar member paths against the destination directory — strip `..` components before extraction
- Add path validation to archive extraction handlers — reject entries that escape the target directory
</patterns>

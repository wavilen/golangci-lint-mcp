# staticcheck: S1014

<instructions>
Detects `append` calls with no additional elements appended, such as `append(s)` or `append(s)`. This is a no-op — it returns the same slice without modification.

Remove the unnecessary append call entirely.
</instructions>

<examples>
## Bad
```go
items = append(items)
```

## Good
```go
// Just use items directly — append with no new elements is a no-op
```
</examples>

<patterns>
- Calling append with no variadic arguments
- Assigning append(s) result back to the same variable
- Leftover debug code or partial refactoring remnants
- Accidental empty append after removing elements
</patterns>

<related>
S1011, S1001, gocritic/appendAssign

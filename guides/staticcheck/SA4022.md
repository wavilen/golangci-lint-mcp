# staticcheck: SA4022

<instructions>
Detects `sort.Slice` calls where the less function is not a valid strict weak ordering. A broken less function can cause `sort.Slice` to produce incorrect results, panic, or enter infinite loops. Common mistakes include using `<=` instead of `<`, or having inconsistent ordering.

The less function must return `true` when `a` should sort before `b`, must be antisymmetric (`less(a,b)` implies `!less(b,a)`), and transitive.
</instructions>

<examples>
## Bad
```go
sort.Slice(items, func(i, j int) bool {
    return items[i].Priority <= items[j].Priority // <= breaks strict ordering
})
```

## Good
```go
sort.Slice(items, func(i, j int) bool {
    return items[i].Priority < items[j].Priority
})
```
</examples>

<patterns>
- Using `<=` or `>=` instead of `<` or `>` in the less function
- Less function that returns true for both `less(a,b)` and `less(b,a)`
- Comparing floating-point values without handling NaN
- Inconsistent tie-breaking when multiple sort keys are involved
</patterns>

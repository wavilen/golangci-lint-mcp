# staticcheck: SA1024

<instructions>
Detects `strings.Trim` (or `TrimLeft`, `TrimRight`) called with a cutset string containing duplicate characters. `Trim` treats the cutset as a set of individual characters, not a substring. Passing `"{}"` when you mean to trim `"{"` and `"}"` separately works, but passing `"{{}}"` is redundant and suggests confusion with substring trimming.
</instructions>

<examples>
## Bad
```go
// Cutset has duplicates — suggests misunderstanding of how Trim works
result := strings.Trim(s, " \t\n\r \t\n\r")
```

## Good
```go
result := strings.Trim(s, " \t\n\r")
// or for whitespace, use the stdlib constant:
result := strings.Trim(s, " ")
```
</examples>

<patterns>
- Passing repeated characters in the cutset to `strings.Trim`
- Confusing `strings.Trim(s, "prefix")` with `strings.TrimPrefix(s, "prefix")`
- Including characters that don't appear in the input string
</patterns>

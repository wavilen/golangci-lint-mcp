# G302

<instructions>
Detects files created or chmod'd with overly permissive permissions. Setting `0666` or world-readable/writable modes allows any local user to read or modify sensitive files.

Use restrictive file permissions. `0600` restricts to owner read/write only; `0640` allows group read. Never use `0666` or similar for files containing sensitive data.
</instructions>

<examples>
## Bad
```go
os.WriteFile("config.json", data, 0666)
os.Chmod("secret.key", 0777)
```

## Good
```go
os.WriteFile("config.json", data, 0600)
os.Chmod("secret.key", 0400)
```
</examples>

<patterns>
- Use restrictive file permissions (`0600` for secrets, `0644` for data) instead of `0666`/`0777`
- Set `os.OpenFile` mode to `0600` for files containing secrets, keys, or credentials
- Set explicit file permissions after creation — do not rely on umask for sensitive files
</patterns>

# G108

<instructions>
Detects when `net/http/pprof` is imported, which automatically registers profiling endpoints (`/debug/pprof/`) on the default HTTP serve mux. In production, this exposes runtime profiling data (goroutine stacks, memory, CPU profiles) to anyone who can reach the server.

Register pprof handlers on a separate mux bound to an internal-only interface, or remove the import entirely for production builds.
</instructions>

<examples>
## Bad
```go
import _ "net/http/pprof"

func main() {
    http.ListenAndServe(":8080", nil) // pprof exposed on public interface
}
```

## Good
```go
func main() {
    // Main server on public interface
    go http.ListenAndServe(":8080", nil)

    // Debug server on localhost only
    mux := http.NewServeMerve()
    mux.HandleFunc("/debug/pprof/", pprof.Index)
    mux.HandleFunc("/debug/pprof/profile", pprof.Profile)
    http.ListenAndServe("127.0.0.1:6060", mux)
}
```
</examples>

<patterns>
- Blank import of `net/http/pprof` alongside `http.ListenAndServe` on `nil` mux
- Profiling endpoints accessible on public-facing ports
- Debug endpoints left enabled in production deployments
</patterns>

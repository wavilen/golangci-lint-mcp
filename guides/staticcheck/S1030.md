# staticcheck: S1030

<instructions>
Detects uses of `fmt.Sprintf` to convert numeric values to strings when `strconv` functions are more efficient and clearer. `strconv.Itoa`, `strconv.FormatFloat`, and `strconv.FormatInt` avoid the overhead of format parsing.

Use the appropriate `strconv` function instead of `fmt.Sprintf`.
</instructions>

<examples>
## Bad
```go
s := fmt.Sprintf("%d", count)
```

## Good
```go
s := strconv.Itoa(count)
```
</examples>

<patterns>
- Using fmt.Sprintf("%d", n) — use strconv.Itoa
- Using fmt.Sprintf("%f", f) — use strconv.FormatFloat
- Using fmt.Sprintf("%x", n) — use strconv.FormatInt with base 16
- Using fmt.Sprintf for any simple numeric-to-string conversion
</patterns>

# staticcheck: SA4027

<instructions>
Detects calls to `net.Conn.Write` (or `WriteTo`, `ReadFrom`) where the return value — specifically the error — is not checked. Network writes can fail silently if the error is ignored, leading to data loss or incomplete communication.

Check the error return value. At minimum, log the error. For critical writes, retry or propagate the error.
</instructions>

<examples>
## Bad
```go
conn.Write(data) // error ignored, data may not be sent
```

## Good
```go
if _, err := conn.Write(data); err != nil {
    return errors.Wrap(err, "write failed")
}
```
</examples>

<patterns>
- `conn.Write(data)` without checking error
- `conn.WriteTo`, `conn.ReadFrom` return values discarded
- HTTP response writes with ignored errors
- File writes in network handlers with unchecked errors
</patterns>

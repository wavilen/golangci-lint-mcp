# staticcheck: ST1000

<instructions>
Detects package comments that do not follow the standard format. Package-level comments should begin with "Package packagename" on the first line. This convention ensures godoc and other documentation tools properly parse the package description.

Rewrite the package comment to start with "Package X" where X is the package name.
</instructions>

<examples>
## Bad
```go
// This package provides utilities for string manipulation.
package strings
```

## Good
```go
// Package strings provides utilities for string manipulation.
package strings
```
</examples>

<patterns>
- Package comments not starting with "Package packagename"
- Using "This package" or "The package" instead of "Package"
- Missing package-level doc comment entirely
- Package comment separated from the package clause by blank lines
</patterns>

<related>

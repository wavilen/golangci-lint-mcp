# staticcheck: SA3000

<instructions>
Detects a `TestMain` function that does not call `os.Exit`. When you define `TestMain`, Go no longer automatically exits the process — you must call `os.Exit` with the return code from `m.Run()`. Without it, the test binary returns exit code 0 even if tests fail.
</instructions>

<examples>
## Good
```go
func TestMain(m *testing.M) {
    setup()
    code := m.Run()
    teardown()
    os.Exit(code)
}
```
</examples>

<patterns>
- Wrap `TestMain` with `os.Exit(m.Run())` to propagate the exit code — do not call `m.Run()` without capturing its result
- Use `os.Exit(m.Run())` in `TestMain` with setup/teardown — returning normally skips the exit code
- Add `os.Exit(m.Run())` when `TestMain` is used to set up global test state
</patterns>

# staticcheck: SA1010

<instructions>
Detects calls to `(*regexp.Regexp).FindAllSubmatch` (and related `FindAll*` methods) where the `n` argument is `0`. When `n` is `0`, the method returns no results — it means "return zero matches," not "return all matches." Pass `-1` to get all matches, or a positive integer to limit the count.
</instructions>

<examples>
## Bad
```go
re := regexp.MustCompile(`(\w+):(\w+)`)
matches := re.FindAllStringSubmatch(input, 0) // returns empty slice always
```

## Good
```go
re := regexp.MustCompile(`(\w+):(\w+)`)
matches := re.FindAllStringSubmatch(input, -1) // returns all matches
```
</examples>

<patterns>
- Replace `n=0` with `n=-1` in `FindAll` methods to get all matches
- Fix the assumption that `n=0` means unlimited — it returns nothing
- Distinguish the `n` parameter in `FindAll` from slice length semantics
</patterns>

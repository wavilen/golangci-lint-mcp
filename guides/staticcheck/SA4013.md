# staticcheck: SA4013

<instructions>
Detects assignments to `err` where the previous error value is discarded without being checked. This means the first error is silently swallowed, which can hide failures. Each assignment to `err` should be preceded by handling the previous value.

Check or return the previous error before overwriting it. Use `err = errors.Join(err, newErr)` to accumulate errors, or use a different variable name.
</instructions>

<examples>
## Good
```go
if _, err := os.Stat(path1); err != nil {
    return err
}
if _, err := os.Stat(path2); err != nil {
    return err
}
```
</examples>

<patterns>
- Check or return each error before overwriting `err` with a new call
- Use `errors.Join` to combine errors instead of overwriting in cleanup blocks
- Handle each error from sequential operations — do not check only the last
- Return early after detecting an error instead of continuing
</patterns>

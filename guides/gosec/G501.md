# G501

<instructions>
Detects imports of `crypto/md5`. MD5 is a broken hash function with known collision attacks — two different inputs can produce the same hash. It should never be used for security-sensitive purposes like signatures, integrity verification, or password hashing.

Use `crypto/sha256` or `crypto/sha512` for hashing. For password storage, use `golang.org/x/crypto/bcrypt` or `argon2`. If you need MD5 only for non-cryptographic checksums (e.g., file deduplication), consider documenting that intent explicitly.
</instructions>

<examples>
## Bad
```go
import "crypto/md5"

func hashData(data []byte) []byte {
    h := md5.Sum(data)
    return h[:]
}
```

## Good
```go
import "crypto/sha256"

func hashData(data []byte) []byte {
    h := sha256.Sum256(data)
    return h[:]
}
```
</examples>

<patterns>
- `crypto/md5` imported for any hashing purpose
- MD5 used in checksums where collision resistance matters
- Legacy systems still relying on MD5 for integrity verification
- Non-cryptographic uses (etag generation, cache keys) that could be replaced
</patterns>

# staticcheck: SA1002

<instructions>
Detects invalid format strings passed to `time.Parse`. The second argument must be a Go reference time layout using `Mon Jan 2 15:04:05 MST 2006` (or `MST -0700` for timezone), not a `time.Format` layout. Mixing up the layout direction or using incorrect component ordering causes this check to fire.

Use the correct Go reference time components. Remember: month is `01`, day is `02`, hour is `15` (not `03`), minute is `04`, second is `05`, year is `2006`.
</instructions>

<examples>
## Bad
```go
// Wrong: uses strftime-style or custom format instead of Go reference time
t, err := time.Parse("YYYY-MM-DD", "2024-01-15")
```

## Good
```go
t, err := time.Parse("2006-01-02", "2024-01-15")
```
</examples>

<patterns>
- Using strftime-style `%Y-%m-%d` or `YYYY-MM-DD` formats instead of Go's `2006-01-02`
- Swapping month and day layout numbers (`02-01` vs `01-02`)
- Using `03:04:05` (12-hour) when `15:04:05` (24-hour) is intended
</patterns>

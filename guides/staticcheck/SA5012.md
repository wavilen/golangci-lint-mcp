# staticcheck: SA5012

<instructions>
Detects empty `if` or `else` branches, such as `if cond {} else { doSomething() }`. An empty branch with no statements is usually a sign of incomplete code or logic that should be inverted.

Invert the condition and remove the empty branch: `if !cond { doSomething() }`.
</instructions>

<examples>
## Bad
```go
if !found {
} else {
    process(item)
}
```

## Good
```go
if found {
    process(item)
}
```
</examples>

<patterns>
- Empty `if` body with logic only in `else`
- Empty `else` body after refactoring
- Placeholder branches not yet implemented
- Negated conditions with empty then-branch
</patterns>
